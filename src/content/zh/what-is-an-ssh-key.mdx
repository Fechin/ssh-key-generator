# SSH 密钥是什么？公钥、私钥与密钥对完整说明

SSH 密钥是一种基于非对称密码学的数字凭证，用于向 SSH 服务器证明您的身份。与传统密码相比，SSH 密钥更长、更随机、不可猜测，并且可以实现无需输入密码的自动化认证。

每位使用 GitHub、GitLab、远程服务器或云基础设施的开发人员都应该了解 SSH 密钥的工作原理，以及如何安全地使用它们。

## SSH 密钥的工作原理

SSH 密钥基于**非对称加密**（也称公钥密码学）。与传统的对称加密使用同一个密钥加密和解密不同，非对称加密使用**一对数学相关但不同**的密钥：公钥和私钥。

这两个密钥有一个关键性质：用公钥加密的数据，只有对应的私钥才能解密；反之，用私钥签名的数据，可以用公钥验证签名的真实性。

### SSH 认证流程

当您使用 SSH 密钥连接服务器时，认证过程如下：

```
1. 客户端向服务器发送"我要使用密钥认证"请求
2. 服务器查找 ~/.ssh/authorized_keys 中对应的公钥
3. 服务器生成一个随机挑战值，用公钥加密后发给客户端
4. 客户端用私钥解密挑战值，计算响应并发回服务器
5. 服务器验证响应正确，认证成功，建立加密会话
```

整个过程中，私钥**从未离开您的计算机**。服务器只需要公钥，无需知道私钥，这使得即使服务器被攻破，您的私钥也不会泄露。

## 公钥与私钥：有什么区别？

理解公钥和私钥的区别是正确使用 SSH 密钥的基础。

### 公钥

公钥可以、也应该自由分享。它的特点是：

- 外观类似：`ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... user@hostname`
- 存储在远程服务器的 `~/.ssh/authorized_keys` 文件中
- 可以上传到 GitHub、GitLab、云服务商等平台
- 即使被他人知晓，也不会带来安全风险
- 通常以 `.pub` 扩展名保存（如 `id_ed25519.pub`）

### 私钥

私钥必须严格保密。它的特点是：

- 始终保存在您的本地计算机上，绝不上传
- 文件权限必须设置为 `600`（仅所有者可读写）
- 可以用密码短语（passphrase）加密保护
- 通常保存为 `id_ed25519`、`id_rsa` 等文件（无扩展名）
- 如果泄露，任何拥有该文件的人都可以冒充您的身份

私钥的安全性决定了整个密钥对的安全性。**如果私钥丢失或泄露，必须立即撤销服务器上对应的公钥，并生成新的密钥对。**

### 形象比喻

可以将 SSH 密钥对想象成**挂锁与钥匙**的关系：
- 公钥就是挂锁，您可以把它挂在任何需要的地方（服务器、GitHub 等）
- 私钥就是打开这把挂锁的唯一钥匙，必须妥善保管

## SSH 密钥算法：Ed25519 与 RSA

不同的 SSH 密钥算法在安全性、性能和兼容性上各有权衡。以下是主要算法的对比：

| 算法 | 密钥长度 | 安全性 | 速度 | 兼容性 | 推荐 |
|------|----------|--------|------|--------|------|
| Ed25519 | 256 位 | 极高 | 极快 | 较新系统 | ✓ 首选 |
| RSA | 2048–4096 位 | 高（4096 位） | 较慢 | 极佳 | 兼容性首选 |
| ECDSA | 256–521 位 | 高 | 快 | 良好 | 可用 |
| DSA | 1024 位 | 低（已废弃） | 中 | 旧系统 | ✗ 不推荐 |

### Ed25519：现代首选

Ed25519 基于 Edwards-curve Digital Signature Algorithm（爱德华兹曲线数字签名算法），使用 Curve25519 椭圆曲线：

```bash
# 生成 Ed25519 密钥（推荐）
ssh-keygen -t ed25519 -C "your_email@example.com"
```

**Ed25519 的优势：**
- 256 位密钥提供约 128 位的安全性，超越 3072 位 RSA
- 签名和验证速度是 RSA 的 10-20 倍
- 密钥文件极小，便于存储和传输
- 设计上避免了许多侧信道攻击
- GitHub、GitLab、主流 Linux 发行版均支持

### RSA：最大兼容性

RSA 是最古老、最广泛支持的 SSH 密钥算法，适合需要连接旧系统的场景：

```bash
# 生成 4096 位 RSA 密钥
ssh-keygen -t rsa -b 4096 -C "your_email@example.com"
```

如果需要连接较旧的 SSH 服务器（如旧版 Cisco 设备、某些嵌入式系统），RSA 可能是唯一选择。对于现代系统，建议优先使用 Ed25519。

## 如何将 SSH 密钥添加到 GitHub

GitHub 使用 SSH 密钥进行 Git 操作认证。添加步骤如下：

### 第一步：生成 SSH 密钥

```bash
ssh-keygen -t ed25519 -C "your_email@github.com"
```

按提示操作，可以使用默认文件路径，并设置一个密码短语（强烈建议）。

### 第二步：复制公钥内容

```bash
# macOS
cat ~/.ssh/id_ed25519.pub | pbcopy

# Linux（安装了 xclip）
cat ~/.ssh/id_ed25519.pub | xclip -selection clipboard

# Windows（PowerShell）
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard
```

或直接查看并手动复制：

```bash
cat ~/.ssh/id_ed25519.pub
```

### 第三步：在 GitHub 添加公钥

1. 登录 GitHub，点击右上角头像
2. 选择 **Settings（设置）**
3. 在左侧菜单选择 **SSH and GPG keys**
4. 点击 **New SSH key**
5. 填写标题（如"我的 MacBook"），粘贴公钥内容
6. 点击 **Add SSH key**

### 第四步：测试连接

```bash
ssh -T git@github.com
# 预期输出：Hi username! You've successfully authenticated...
```

此后，使用 SSH URL 克隆仓库即可免密码推送：

```bash
git clone git@github.com:username/repository.git
```

了解更多 SSH 配置详情，请参阅<a href="/zh-Hans/how-to-set-up-ssh/">如何设置 SSH：完整分步指南</a>。

## SSH 密钥存储在哪里？

SSH 密钥默认存储在用户主目录下的 `.ssh` 隐藏目录中：

### Linux 和 macOS

| 文件 | 说明 |
|------|------|
| `~/.ssh/id_ed25519` | Ed25519 私钥 |
| `~/.ssh/id_ed25519.pub` | Ed25519 公钥 |
| `~/.ssh/id_rsa` | RSA 私钥 |
| `~/.ssh/id_rsa.pub` | RSA 公钥 |
| `~/.ssh/authorized_keys` | 服务器上存储已授权公钥的文件 |
| `~/.ssh/known_hosts` | 已知服务器的主机密钥记录 |
| `~/.ssh/config` | SSH 客户端配置文件 |

检查现有密钥：

```bash
ls -la ~/.ssh/
```

### Windows

在 Windows 上，SSH 密钥默认存储在：

```
C:\Users\YourUsername\.ssh\
```

使用 PowerShell 查看：

```powershell
Get-ChildItem ~/.ssh/
```

### 权限设置

SSH 对文件权限有严格要求，错误的权限会导致认证失败：

```bash
# 设置 .ssh 目录权限
chmod 700 ~/.ssh

# 设置私钥权限（仅所有者可读写）
chmod 600 ~/.ssh/id_ed25519

# 设置公钥权限
chmod 644 ~/.ssh/id_ed25519.pub

# 设置 authorized_keys 权限
chmod 600 ~/.ssh/authorized_keys
```

## SSH 密钥密码短语

生成 SSH 密钥时，系统会提示设置**密码短语（passphrase）**。这是对私钥文件的额外加密保护。

### 密码短语的作用

私钥文件加密存储在您的磁盘上。即使有人获取了您的私钥文件，没有密码短语也无法使用它。这提供了**第二层安全保护**：

- **无密码短语**：私钥文件即可直接用于认证，适合自动化场景
- **有密码短语**：每次使用私钥时需要输入密码短语，更安全

### 使用 ssh-agent 避免重复输入

`ssh-agent` 可以在内存中缓存已解密的私钥，避免每次 SSH 连接都输入密码短语：

```bash
# 启动 ssh-agent
eval "$(ssh-agent -s)"

# 将私钥添加到 agent（会提示输入密码短语）
ssh-add ~/.ssh/id_ed25519

# 查看已加载的密钥
ssh-add -l

# macOS：持久化到 Keychain（重启后也无需重新输入）
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
```

### 更改密码短语

您可以在不更改密钥对的情况下更改密码短语：

```bash
ssh-keygen -p -f ~/.ssh/id_ed25519
```

## SSH 密钥指纹

SSH 密钥指纹（fingerprint）是对公钥内容进行哈希运算后得到的简短字符串，用于唯一标识一个密钥，便于人工验证。

### 查看密钥指纹

```bash
# 查看默认格式（SHA-256，Base64 编码）
ssh-keygen -lf ~/.ssh/id_ed25519.pub

# 查看 MD5 格式（某些旧系统使用）
ssh-keygen -l -E md5 -f ~/.ssh/id_ed25519.pub
```

输出示例：

```
256 SHA256:abc123...XYZ user@hostname (ED25519)
```

### 服务器主机密钥验证

首次连接新服务器时，SSH 会显示服务器的主机密钥指纹并询问是否信任：

```
The authenticity of host 'example.com (1.2.3.4)' can't be established.
ED25519 key fingerprint is SHA256:abc123...XYZ.
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

最佳实践是通过**带外渠道**（如云控制台、服务器配置文档）验证指纹后再确认。确认后，服务器指纹会保存到 `~/.ssh/known_hosts`，以后连接时自动验证，防止中间人攻击。

## 常见问题解答

### SSH 密钥有什么用途？

SSH 密钥用于向 SSH 服务器证明您的身份，无需输入密码。主要用途包括：安全登录远程服务器、向 GitHub/GitLab 推送代码、自动化部署脚本、SFTP/SCP 文件传输，以及 CI/CD 流水线中的自动化操作。

### 我应该有多少个 SSH 密钥？

这取决于您的安全策略。常见做法是：**每台设备一个密钥**（如工作电脑一个、家用电脑一个），这样如果某台设备丢失，可以精准撤销对应的公钥而不影响其他设备。也有人为不同用途（如 GitHub 专用、服务器专用）创建独立的密钥对，以便更精细地管理权限。

### 我可以共享我的 SSH 公钥吗？

可以，SSH **公钥**可以安全地共享。您可以把公钥发给服务器管理员、上传到 GitHub/GitLab、放在公司的 SSH 授权系统中。但**私钥**绝对不能共享，即使是与系统管理员或同事。

### 如果丢失了 SSH 私钥会怎样？

如果丢失了私钥，您将无法再使用该密钥对进行认证。您需要：首先通过其他方式（控制台访问、备用密钥）登录服务器，从 `~/.ssh/authorized_keys` 中删除对应的旧公钥，然后生成新的密钥对，并将新公钥添加到所有相关服务器和平台。这正是为什么建议定期备份密钥，或为每个重要服务维护备用访问方式。

### Ed25519 和 RSA 哪个更好？

对于**现代系统**，Ed25519 更好：密钥更短、速度更快、安全性更高。GitHub、GitLab 和主流 Linux 发行版都支持 Ed25519。只有在需要连接**旧系统**（如某些网络设备、旧版 OpenSSH）时，才需要使用 RSA。如果不确定，优先选择 Ed25519，连接失败再改用 RSA 4096。
