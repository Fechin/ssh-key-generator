# ssh-keygen-kommando: skapa SSH-nycklar med exempel

Praktiska ssh-keygen-exempel för Ed25519- och RSA-nycklar, filnamn, lösenfras, fingeravtryck, GitHub och serverinloggning.

Att skapa en SSH-nyckel betyder att skapa ett par med publik och privat nyckel. För nya nycklar är Ed25519 oftast rätt val; RSA 4096 behövs främst för äldre system.

## Snabbt svar

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
ssh-keygen -lf ~/.ssh/id_ed25519.pub
```

## Grundsyntax för ssh-keygen

```bash
ssh-keygen -t TYPE -b BITS -C COMMENT -f OUTPUT_FILE
```

- `-t`: nyckeltyp
- `-b`: RSA-bitstorlek
- `-C`: kommentar
- `-f`: utdatafil
- `-p`: ändra lösenfras
- `-l`: fingeravtryck
- `-y`: hämta publik nyckel från privat nyckel

## Ed25519

För nya nycklar väljer du oftast Ed25519. Använd RSA 4096 när en äldre SSH-server eller ett verktyg saknar stöd för Ed25519.

```bash
ssh-keygen -t ed25519 -C "you@example.com"
```

## RSA 4096

För nya nycklar väljer du oftast Ed25519. Använd RSA 4096 när en äldre SSH-server eller ett verktyg saknar stöd för Ed25519.

```bash
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

## Filnamn, fingeravtryck och ssh-agent

De flesta anslutningar använder ssh user@host. Använd -i för en viss nyckel, -p för annan port och -J för anslutning via bastion.

```bash
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
ssh-keygen -p -f ~/.ssh/id_ed25519
ssh-keygen -lf ~/.ssh/id_ed25519.pub
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## Lägg till nyckeln i GitHub, GitLab eller på en server

Vanligt flöde: skapa nyckeln, kopiera den publika nyckeln, lägg till den i GitHub/GitLab eller på servern, ladda den privata nyckeln i ssh-agent och testa anslutningen.

```bash
ssh -T git@github.com
ssh -v user@server.example.com
```
