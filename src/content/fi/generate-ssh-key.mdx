# Luo SSH-avain: verkkotyökalu ja ssh-keygen-komennot

SSH-avaimen luonti tarkoittaa julkisen ja yksityisen avaimen parin tekemistä. Uusille avaimille Ed25519 on yleensä paras valinta; RSA 4096 on lähinnä vanhoja järjestelmiä varten.

Verkkogeneraattori luo avainparin selaimessa. Julkinen avain lisätään GitHubiin, GitLabiin, Bitbucketiin tai palvelimelle; yksityinen avain jää omalle koneellesi.

## Nopea vastaus

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

<a href="/fi/">SSH-avaingeneraattori: luo SSH-avaimia verkossa</a> Luo Ed25519- tai RSA-SSH-avaimia selaimessa GitHubia, GitLabia, palvelimia ja ssh-keygeniä varten. Yksityistä avainta ei lähetetä.

## Luo SSH-avain verkossa

1. Avaa SSH-avaingeneraattori.
2. Valitse Ed25519 moderneille järjestelmille tai RSA 4096 vanhaa yhteensopivuutta varten.
3. Lisää selkeä kommentti, kuten sähköposti tai projektin nimi.
4. Käytä henkilökohtaisille avaimille salalausetta.
5. Luo avainpari ja kopioi tai lataa tiedostot.
6. Asenna vain julkinen avain GitHubiin, GitLabiin, Bitbucketiin tai palvelimelle.

Julkisen avaimen voi jakaa. Yksityinen avain on pidettävä salassa, eikä sitä pidä liittää tiketteihin, chatteihin tai dokumentaatioon.

## Luo SSH-avain ssh-keygenillä

`ssh-keygen` / OpenSSH / Linux / macOS / Windows

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
```

## Lisää avain GitHubiin, GitLabiin tai palvelimelle

Tavallinen työnkulku: luo avain, kopioi julkinen avain, lisää se GitHubiin/GitLabiin tai palvelimelle, lataa yksityinen avain ssh-agenttiin ja testaa yhteys.

```bash
ssh -T git@github.com
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com
```

## Valitse Ed25519 tai RSA

| Käyttö | Suositeltu avain |
|---|---|
| GitHub, GitLab, Bitbucket | Ed25519 |
| Moderni Linux-palvelin | Ed25519 |
| CI/CD-deploy-avain | Ed25519 |
| Vanha SSH-palvelin | RSA 4096 |

Valitse uusille avaimille yleensä Ed25519. Käytä RSA 4096:ta, jos vanha SSH-palvelin tai työkalu ei tue Ed25519:ää.

## Salalause ja ssh-agent

Julkisen avaimen voi jakaa. Yksityinen avain on pidettävä salassa, eikä sitä pidä liittää tiketteihin, chatteihin tai dokumentaatioon.

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## Yleinen vianmääritys

Useimpiin yhteyksiin riittää ssh user@host. Käytä -i:tä tietylle avaimelle, -p:tä eri portille ja -J:tä bastionin kautta yhdistämiseen.

```bash
ssh -v user@server.example.com
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
ssh -i ~/.ssh/id_ed25519_work user@server.example.com
```
