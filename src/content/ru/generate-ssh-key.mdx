# Создать SSH-ключ: онлайн-генератор и команды ssh-keygen

Создать SSH-ключ значит получить пару из открытого и приватного ключа. Для новых ключей обычно выбирают Ed25519; RSA 4096 нужен для старых серверов или явных требований.

Онлайн-генератор SSH-ключей создаёт пару прямо в браузере. Открытый ключ добавляют в GitHub, GitLab, Bitbucket или на сервер; приватный ключ остаётся на вашем компьютере.

## Короткий ответ

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

<a href="/ru/">Генератор SSH-ключей: создать SSH-ключ онлайн</a> Создавайте Ed25519 или RSA SSH-ключи в браузере для GitHub, GitLab, серверов и ssh-keygen. Приватный ключ не загружается.

## Создать SSH-ключ онлайн

1. Откройте генератор SSH-ключей.
2. Выберите Ed25519 для современных систем или RSA 4096 для совместимости со старыми окружениями.
3. Добавьте понятный комментарий: email или название проекта.
4. Для личных ключей задайте passphrase.
5. Создайте пару ключей и сохраните открытый и приватный ключи.
6. Добавляйте на GitHub, GitLab, Bitbucket или сервер только открытый ключ.

Открытый ключ можно передавать. Приватный ключ должен оставаться секретным: не вставляйте его в тикеты, чаты и документацию.

## Создать SSH-ключ через ssh-keygen

`ssh-keygen` / OpenSSH / Linux / macOS / Windows

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
```

## Добавить ключ в GitHub, GitLab или на сервер

Обычный порядок: создать ключ, скопировать открытый ключ, добавить его в GitHub/GitLab или на сервер, загрузить приватный ключ в ssh-agent и проверить подключение.

```bash
ssh -T git@github.com
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com
```

## Выбор между Ed25519 и RSA

| Сценарий | Рекомендуемый ключ |
|---|---|
| GitHub, GitLab, Bitbucket | Ed25519 |
| Современный Linux-сервер | Ed25519 |
| Ключ для CI/CD | Ed25519 |
| Старый SSH-сервер | RSA 4096 |

Для новых ключей обычно подходит Ed25519. Используйте RSA 4096, если старый SSH-сервер или инструмент не поддерживает Ed25519.

## Passphrase и ssh-agent

Открытый ключ можно передавать. Приватный ключ должен оставаться секретным: не вставляйте его в тикеты, чаты и документацию.

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## Типичные проблемы

Для большинства подключений достаточно ssh user@host. Используйте -i для выбора ключа, -p для нестандартного порта и -J для подключения через бастион.

```bash
ssh -v user@server.example.com
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
ssh -i ~/.ssh/id_ed25519_work user@server.example.com
```
