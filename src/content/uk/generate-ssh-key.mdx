# Створити SSH-ключ: онлайн-генератор і команди ssh-keygen

Створити SSH-ключ означає отримати пару з відкритого та приватного ключа. Для нових ключів зазвичай обирають Ed25519; RSA 4096 потрібен для старих серверів або явних вимог.

Онлайн-генератор створює пару ключів прямо в браузері. Відкритий ключ додають у GitHub, GitLab, Bitbucket або на сервер; приватний ключ залишається на вашому комп’ютері.

## Коротка відповідь

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

<a href="/uk/">Генератор SSH-ключів: створити SSH-ключ онлайн</a> Створюйте Ed25519 або RSA SSH-ключі в браузері для GitHub, GitLab, серверів і ssh-keygen. Приватний ключ не завантажується.

## Створити SSH-ключ онлайн

1. Відкрийте генератор SSH-ключів.
2. Оберіть Ed25519 для сучасних систем або RSA 4096 для сумісності зі старими середовищами.
3. Додайте зрозумілий коментар: email або назву проєкту.
4. Для особистих ключів задайте passphrase.
5. Створіть пару ключів і збережіть відкритий та приватний ключі.
6. Додавайте до GitHub, GitLab, Bitbucket або на сервер лише відкритий ключ.

Відкритим ключем можна ділитися. Приватний ключ має залишатися секретним: не вставляйте його в тікети, чати чи документацію.

## Створити SSH-ключ через ssh-keygen

`ssh-keygen` / OpenSSH / Linux / macOS / Windows

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
```

## Додати ключ до GitHub, GitLab або на сервер

Звичайний порядок: створити ключ, скопіювати відкритий ключ, додати його в GitHub/GitLab або на сервер, завантажити приватний ключ у ssh-agent і перевірити підключення.

```bash
ssh -T git@github.com
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com
```

## Вибір між Ed25519 і RSA

| Сценарій | Рекомендований ключ |
|---|---|
| GitHub, GitLab, Bitbucket | Ed25519 |
| Сучасний Linux-сервер | Ed25519 |
| Ключ для CI/CD | Ed25519 |
| Старий SSH-сервер | RSA 4096 |

Для нових ключів зазвичай підходить Ed25519. Використовуйте RSA 4096, якщо старий SSH-сервер або інструмент не підтримує Ed25519.

## Passphrase і ssh-agent

Відкритим ключем можна ділитися. Приватний ключ має залишатися секретним: не вставляйте його в тікети, чати чи документацію.

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## Типові проблеми

Для більшості підключень достатньо ssh user@host. Використовуйте -i для вибору ключа, -p для нестандартного порту і -J для підключення через бастіон.

```bash
ssh -v user@server.example.com
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
ssh -i ~/.ssh/id_ed25519_work user@server.example.com
```
