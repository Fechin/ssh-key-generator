# Πώς να ρυθμίσετε το SSH: Ένας πλήρης οδηγός βήμα προς βήμα για Linux, Mac και Windows

Η σωστή ρύθμιση του SSH διαρκεί περίπου δέκα λεπτά και αποδίδει μερίσματα για χρόνια. Αφού ρυθμίσετε τις παραμέτρους, μπορείτε να συνδεθείτε σε διακομιστές χωρίς να πληκτρολογήσετε κωδικούς πρόσβασης, να προωθήσετε τον κώδικα στο GitHub χωρίς να εισάγετε διαπιστευτήρια σε κάθε λειτουργία και να αυτοματοποιήσετε τις αναπτύξεις με σιγουριά. Αυτός ο οδηγός περιγράφει ολόκληρη τη διαδικασία — από τη δημιουργία ενός ζεύγους κλειδιών έως την αντιμετώπιση προβλημάτων κοινών σφαλμάτων — για Linux, macOS και Windows.

Εάν θέλετε να κατανοήσετε πρώτα την υποκείμενη τεχνολογία, διαβάστε τους οδηγούς μας σχετικά με το <a href="/el/what-is-ssh">τι είναι το SSH</a> και το <a href="/el/what-is-an-ssh-key">τι είναι τα κλειδιά SSH</a>.

## Προαπαιτούμενα

Πριν ξεκινήσετε, βεβαιωθείτε ότι έχετε:

**Για πρόσβαση διακομιστή:**
- Μια εφαρμογή τερματικού (Terminal σε macOS/Linux, Windows Terminal ή PowerShell στα Windows)
- Ένας απομακρυσμένος διακομιστής με Linux/Unix με εγκατεστημένο λογισμικό διακομιστή SSH (sshd).
- Ένας λογαριασμός χρήστη σε αυτόν τον διακομιστή (ή πρόσβαση root για τη δημιουργία ενός)
- Η διεύθυνση IP ή το όνομα κεντρικού υπολογιστή του διακομιστή

**Για πρόσβαση στο GitHub/GitLab:**
- Ένας λογαριασμός GitHub ή GitLab
- Διατίθεται τερματικό με `ssh`
- Προαιρετικά: το git εγκατεστημένο τοπικά

**Απαιτήσεις λειτουργικού συστήματος:**
- **Linux**: Το πρόγραμμα-πελάτης SSH είναι προεγκατεστημένο σχεδόν σε όλες τις διανομές
- **macOS**: Το πρόγραμμα-πελάτης SSH είναι ενσωματωμένο, προσβάσιμο από το τερματικό
- **Windows 10 έκδοση 1809+ / Windows 11**: Ο πελάτης OpenSSH είναι διαθέσιμος ως προαιρετική δυνατότητα

Δεν χρειάζεται να εγκαταστήσετε τίποτα σε Linux ή macOS. Στα Windows, επαληθεύστε ότι το OpenSSH είναι εγκατεστημένο ανοίγοντας το PowerShell και εκτελώντας το `ssh -V`. Εάν λάβετε μια συμβολοσειρά έκδοσης, είστε έτοιμοι. Εάν όχι, ανατρέξτε στο Βήμα 4 για οδηγίες εγκατάστασης.

## Βήμα 1: Δημιουργήστε το ζεύγος κλειδιών SSH

Χρειάζεστε ένα ζεύγος κλειδιών για να μπορέσετε να κάνετε οτιδήποτε άλλο. Το ζεύγος κλειδιών αποτελείται από ένα **ιδιωτικό κλειδί** (παραμένει στον υπολογιστή σας) και ένα **δημόσιο κλειδί** (μεταφέρεται σε διακομιστές ή υπηρεσίες όπως το GitHub).

### Επιλογή Α: Χρησιμοποιήστε το Online Generator

Εάν προτιμάτε μια γραφική διεπαφή, η <a href="/">δωρεάν ηλεκτρονική μας δημιουργία κλειδιών SSH</a> δημιουργεί ζεύγη κλειδιών Ed25519 ή RSA στο πρόγραμμα περιήγησής σας. Το ιδιωτικό κλειδί δημιουργείται τοπικά χρησιμοποιώντας το Web Crypto API — τίποτα δεν αποστέλλεται στους διακομιστές μας. Κατεβάστε και τα δύο αρχεία και αποθηκεύστε τα στο `~/.ssh/` (Linux/macOS) ή στο `C:\Users\YourName\.ssh\` (Windows).

### Επιλογή Β: Χρησιμοποιήστε το ssh-keygen στη γραμμή εντολών

Η εντολή `ssh-keygen` είναι διαθέσιμη σε Linux, macOS και Windows με OpenSSH:

```bash
# Δημιουργήστε ένα ζεύγος κλειδιών Ed25519 (συνιστάται)
ssh-keygen -t ed25519 -C "your@email.com"
```

**Τι σημαίνει κάθε σημαία:**
- `-t ed25519` — τύπος κλειδιού (το Ed25519 είναι σύγχρονο, γρήγορο και ασφαλές)
- `-C "comment"` — μια ετικέτα ενσωματωμένη στο δημόσιο κλειδί, συνήθως τη διεύθυνση email σας

Θα σας ζητηθεί για:

```
Generating public/private ed25519 key pair.
Enter file in which to save the key (/home/user/.ssh/id_ed25519):
```

Πατήστε **Enter** για να αποδεχτείτε την προεπιλεγμένη τοποθεσία. Εάν έχετε πολλά κλειδιά, μπορείτε να τα ονομάσετε περιγραφικά (π.χ. `/home/user/.ssh/id_ed25519_github`).

```
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
```

Εισαγάγετε μια ισχυρή φράση πρόσβασης. Αυτό κρυπτογραφεί το αρχείο του ιδιωτικού κλειδιού στο δίσκο — εάν το μηχάνημά σας κλαπεί, το κλειδί δεν μπορεί να χρησιμοποιηθεί χωρίς τη φράση πρόσβασης. Μπορείτε να το αφήσετε κενό για περιπτώσεις χρήσης αυτοματισμού όπου δεν είναι δυνατή η ανθρώπινη αλληλεπίδραση, αλλά για προσωπικά κλειδιά, χρησιμοποιείτε πάντα μια φράση πρόσβασης.

**Παραγωγή:**

```
Your identification has been saved in /home/user/.ssh/id_ed25519
Your public key has been saved in /home/user/.ssh/id_ed25519.pub
The key fingerprint is:
SHA256:abc123defghijk... your@email.com
The key's randomart image is:
+--[ED25519 256]--+
...
```

Δημιουργούνται δύο αρχεία:
- `~/.ssh/id_ed25519` — το **προσωπικό σας κλειδί** (μην το κοινοποιείτε ποτέ)
- `~/.ssh/id_ed25519.pub` — το **δημόσιο κλειδί σας** (ασφαλές για κοινή χρήση)

### Για RSA (Συμβατότητα παλαιού τύπου)

Εάν πρέπει να συνδεθείτε σε πολύ παλιά συστήματα που δεν υποστηρίζουν το Ed25519:

```bash
ssh-keygen -t rsa -b 4096 -C "your@email.com"
```

Χρησιμοποιήστε τουλάχιστον 4096 bit. Τα κλειδιά RSA μικρότερα από 4096 bit δεν θεωρούνται πλέον επαρκής ασφάλεια.

## Βήμα 2: Ρύθμιση SSH σε Linux

Αυτή η ενότητα καλύπτει την αντιγραφή του δημόσιου κλειδιού σας σε διακομιστή Linux, ώστε να μπορείτε να κάνετε έλεγχο ταυτότητας χωρίς κωδικό πρόσβασης.

### Μέθοδος 1: ssh-copy-id (Ευκολότερη)

Το `ssh-copy-id` είναι ο απλούστερος τρόπος για να εγκαταστήσετε το δημόσιο κλειδί σας σε έναν απομακρυσμένο διακομιστή. Συνδέεται χρησιμοποιώντας έλεγχο ταυτότητας με κωδικό πρόσβασης, προσθέτει το δημόσιο κλειδί σας στο `~/.ssh/authorized_keys` του διακομιστή και ορίζει τα σωστά δικαιώματα.

```bash
ssh-copy-id user@server.example.com

# Καθορίστε ρητά ένα αρχείο κλειδιού
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com

# Συνδέστε σε μια μη τυπική θύρα
ssh-copy-id -p 2222 user@server.example.com
```

Θα σας ζητηθεί για τελευταία φορά ο κωδικός πρόσβασης διακομιστή. Μετά από αυτό, μπορείτε να συνδεθείτε με το `ssh user@server.example.com` χρησιμοποιώντας το κλειδί σας.

### Μέθοδος 2: Μη αυτόματη αντιγραφή

Εάν το `ssh-copy-id` δεν είναι διαθέσιμο (συχνό σε ορισμένα συστήματα BSD ή ελάχιστες εγκαταστάσεις Linux):

```bash
# Εμφανίστε το δημόσιο κλειδί σας και αντιγράψτε την έξοδο
cat ~/.ssh/id_ed25519.pub

# Στη συνέχεια, SSH στον διακομιστή (με κωδικό πρόσβασης) και εκτελέστε:
mkdir -p ~/.ssh
chmod 700 ~/.ssh
echo "PASTE_YOUR_PUBLIC_KEY_HERE" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

Ή κάντε τα όλα σε έναν αγωγό:

```bash
cat ~/.ssh/id_ed25519.pub | ssh user@server.example.com \
  "mkdir -p ~/.ssh && chmod 700 ~/.ssh && \
   cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

### Επαλήθευση ότι το κλειδί έχει εγκατασταθεί

```bash
ssh user@server.example.com
# Θα πρέπει να συνδεθεί χωρίς να ζητηθεί κωδικός πρόσβασης (μόνο φράση πρόσβασης εάν ορίσετε έναν)
```

### Διαμόρφωση του διακομιστή SSH (sshd)

Στον απομακρυσμένο διακομιστή, η διαμόρφωση δαίμονα SSH βρίσκεται στο `/etc/ssh/sshd_config`. Για ασφαλή εγκατάσταση:

```bash
sudo nano /etc/ssh/sshd_config
```

Προτεινόμενες ρυθμίσεις:

```
# Ενεργοποίηση ελέγχου ταυτότητας δημόσιου κλειδιού
PubkeyAuthentication yes

# Απενεργοποιήστε τον έλεγχο ταυτότητας με κωδικό πρόσβασης (αφού επιβεβαιώσετε ότι τα κλειδιά λειτουργούν!)
PasswordAuthentication no

# Απενεργοποιήστε τη σύνδεση root μέσω SSH
PermitRootLogin no

# Επιτρέπονται μόνο συγκεκριμένοι χρήστες
AllowUsers alice bob deploy

# Κρατήστε τις συνεδρίες ζωντανές
ClientAliveInterval 60
ClientAliveCountMax 3
```

Μετά την επεξεργασία, φορτώστε ξανά τον δαίμονα SSH:

```bash
sudo systemctl reload sshd
# Ή σε παλαιότερα συστήματα:
sudo service ssh reload
```

**Σημαντικό**: Μην απενεργοποιήσετε το `PasswordAuthentication` έως ότου επιβεβαιώσετε ότι ο έλεγχος ταυτότητας βάσει κλειδιού λειτουργεί. Εάν κλειδωθείτε έξω, θα χρειαστείτε πρόσβαση στην κονσόλα (ή πρόσβαση έκτακτης ανάγκης του παρόχου φιλοξενίας σας) για να ανακτήσετε.

### Ρύθμιση του ssh-agent στο Linux

Εάν το κλειδί σας έχει μια φράση πρόσβασης, προσθέστε τη στο `ssh-agent` στην αρχή της συνεδρίας σας για να αποφύγετε την επανεισαγωγή της:

```bash
# Εκκίνηση του πράκτορα (συχνά εκκινείται αυτόματα από περιβάλλοντα επιφάνειας εργασίας)
eval "$(ssh-agent -s)"

# Προσθέστε το κλειδί σας (εισάγετε μια φράση πρόσβασης)
ssh-add ~/.ssh/id_ed25519

# Επαληθεύστε ότι τα κλειδιά έχουν φορτωθεί
ssh-add -l
```

Για αυτόματη εκκίνηση του `ssh-agent` και προσθήκη του κλειδιού σας κατά τη σύνδεση, προσθέστε στο `~/.bashrc` ή στο `~/.zshrc`:

```bash
# Αυτόματη εκκίνηση του ssh-agent
if [ -z "$SSH_AUTH_SOCK" ]; then
  eval "$(ssh-agent -s)"
  ssh-add ~/.ssh/id_ed25519
fi
```

## Βήμα 3: Ρυθμίστε το SSH στο macOS

Το macOS περιλαμβάνει ένα πλήρες πρόγραμμα-πελάτη OpenSSH. Η διαδικασία για τη ρύθμιση του ελέγχου ταυτότητας βάσει κλειδιού σε έναν απομακρυσμένο διακομιστή είναι πανομοιότυπη με το Linux (βλ. Βήμα 2 παραπάνω). Οι ειδικές για το macOS προσθήκες περιλαμβάνουν ενσωμάτωση Keychain.

### Ενσωμάτωση μπρελόκ

Το macOS μπορεί να αποθηκεύσει τη φράση πρόσβασης στο σύστημα Keychain, ώστε να μην χρειαστεί ποτέ να την εισαγάγετε μετά την πρώτη φορά:

```bash
# Προσθήκη κλειδιού στο ssh-agent με ενσωμάτωση Keychain
ssh-add --apple-use-keychain ~/.ssh/id_ed25519

# Για παλαιότερες εκδόσεις macOS (πριν από το Monterey)
ssh-add -K ~/.ssh/id_ed25519
```

Για να έχετε το macOS να φορτώνει πάντα τα κλειδιά σας από το Keychain, προσθέστε αυτό στο `~/.ssh/config`:

```
Host *
    AddKeysToAgent yes
    UseKeychain yes
    IdentityFile ~/.ssh/id_ed25519
```

Μετά από αυτό, η φράση πρόσβασης αποθηκεύεται στο Keychain και φορτώνεται αυτόματα — δεν θα σας ζητηθεί ξανά στις επόμενες μπότες.

### Επαληθεύστε την έκδοση πελάτη SSH

```bash
ssh -V
# OpenSSH_9.x, LibreSSL x.x.x
```

Το macOS διατίθεται με ένα σχετικά πρόσφατο OpenSSH αλλά όχι πάντα το απόλυτο τελευταίο. Εάν χρειάζεστε νεότερες δυνατότητες, εγκαταστήστε μέσω του Homebrew: `brew install openssh`.

## Βήμα 4: Ρύθμιση SSH στα Windows

Τα Windows 10 (1809+) και τα Windows 11 περιλαμβάνουν το OpenSSH. Εδώ είναι πώς να το κάνετε να λειτουργήσει.

### Εγκαταστήστε το OpenSSH Client

**Ελέγξτε αν είναι ήδη εγκατεστημένο:**

```powershell
ssh -V
```

**Εγκατάσταση μέσω ρυθμίσεων:**
1. Ανοίξτε **Ρυθμίσεις → Εφαρμογές → Προαιρετικές λειτουργίες**
2. Αναζήτηση για **OpenSSH Client**
3. Κάντε κλικ στο **Εγκατάσταση**

**Εγκατάσταση μέσω PowerShell (Διαχειριστής):**

```powershell
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0
```

### Δημιουργία κλειδιών στα Windows

Ανοίξτε το PowerShell και εκτελέστε:

```powershell
ssh-keygen -t ed25519 -C "your@email.com"
```

Τα κλειδιά αποθηκεύονται στο `C:\Users\YourUsername\.ssh\`.

### Αντιγράψτε το δημόσιο κλειδί σας σε διακομιστή Linux

Τα Windows δεν περιλαμβάνουν το `ssh-copy-id`, αλλά μπορείτε να το κάνετε χειροκίνητα από το PowerShell:

```powershell
# Εμφάνιση του δημόσιου κλειδιού σας
Get-Content ~/.ssh/id_ed25519.pub

# Αντιγράψτε το στον απομακρυσμένο διακομιστή (θα σας ζητηθεί ο κωδικός πρόσβασης διακομιστή)
$pubkey = Get-Content ~/.ssh/id_ed25519.pub -Raw
ssh user@server.example.com "mkdir -p ~/.ssh && echo '$pubkey' >> ~/.ssh/authorized_keys && chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys"
```

Ή χρησιμοποιήστε το `Add-Content` για να το διοχετεύσετε:

```powershell
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh user@server.example.com "cat >> ~/.ssh/authorized_keys"
```

### Ρύθμιση του ssh-agent στα Windows

Για να αποφύγετε την επανειλημμένη εισαγωγή της φράσης πρόσβασης, ξεκινήστε την υπηρεσία OpenSSH Authentication Agent:

```powershell
# Ξεκινήστε την υπηρεσία
Start-Service ssh-agent

# Ρυθμίστε το να ξεκινά αυτόματα
Set-Service -Name ssh-agent -StartupType Automatic

# Προσθέστε το κλειδί σας
ssh-add ~/.ssh/id_ed25519
```

### PuTTY ως εναλλακτική λύση

Εάν προτιμάτε ένα πρόγραμμα-πελάτη SSH με γραφικά, το PuTTY είναι μια δωρεάν επιλογή για τα Windows. Χρησιμοποιήστε το **PuTTYgen** για να δημιουργήσετε κλειδιά (αποθήκευση σε μορφή `.ppk` για PuTTY και εξαγωγή σε μορφή OpenSSH). Ωστόσο, για νέες ρυθμίσεις, ο ενσωματωμένος πελάτης OpenSSH είναι απλούστερος και διαλειτουργικός με το υπόλοιπο οικοσύστημα.

## Πώς να ρυθμίσετε τα κλειδιά SSH για το GitHub και το GitLab

Ο έλεγχος ταυτότητας κλειδιού SSH στο GitHub και στο GitLab λειτουργεί με παρόμοιο τρόπο — ανεβάζετε το δημόσιο κλειδί σας στην υπηρεσία και χρησιμοποιείτε το αντίστοιχο ιδιωτικό κλειδί όταν εκτελείτε λειτουργίες Git.

### Ρύθμιση SSH για GitHub

**1. Αντιγράψτε το δημόσιο κλειδί σας:**

```bash
# Linux
cat ~/.ssh/id_ed25519.pub | xclip -selection clipboard

# macOS
cat ~/.ssh/id_ed25519.pub | pbcopy

# Windows PowerShell
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard
```

Ή απλώς εμφανίστε και αντιγράψτε χειροκίνητα:

```bash
cat ~/.ssh/id_ed25519.pub
```

**2. Προσθήκη στο GitHub:**

1. Μεταβείτε στο **github.com** και συνδεθείτε
2. Κάντε κλικ στο avatar σας → **Ρυθμίσεις**
3. Στην αριστερή πλαϊνή γραμμή: **Κλειδιά SSH και GPG**
4. Κάντε κλικ στο **Νέο κλειδί SSH**
5. Δώστε του έναν περιγραφικό **Τίτλο** (π.χ. "MacBook Pro" ή "Work PC")
6. Ορίστε **Τύπος κλειδιού** σε **Κλειδί ελέγχου ταυτότητας**
7. Επικολλήστε το δημόσιο κλειδί σας
8. Κάντε κλικ στο **Προσθήκη κλειδιού SSH**

**3. Δοκιμάστε τη σύνδεση:**

```bash
ssh -T git@github.com
# Γεια όνομα χρήστη! Πραγματοποιήσατε επιτυχή έλεγχο ταυτότητας, αλλά το GitHub δεν παρέχει πρόσβαση στο κέλυφος.
```

**4. Χρήση SSH για λειτουργίες Git:**

```bash
# Κλωνοποίηση χρησιμοποιώντας SSH αντί για HTTPS
git clone git@github.com:username/repository.git

# Εάν ένα υπάρχον repo χρησιμοποιεί HTTPS, αλλάξτε το σε SSH
git remote set-url origin git@github.com:username/repository.git
```

### Ρύθμιση SSH για το GitLab

Η διαδικασία είναι ίδια:

1. Μεταβείτε στο **gitlab.com** → Κάντε κλικ στο avatar σας → **Προτιμήσεις**
2. Στην αριστερή πλαϊνή γραμμή: **Κλειδιά SSH**
3. Επικολλήστε το δημόσιο κλειδί σας, δώστε του έναν τίτλο και ορίστε προαιρετικά μια ημερομηνία λήξης
4. Κάντε κλικ στο **Προσθήκη κλειδιού**

Δοκιμή με:

```bash
ssh -T git@gitlab.com
# Καλώς ήρθατε στο GitLab, @username!
```

Για αυτο-φιλοξενούμενες παρουσίες GitLab, αντικαταστήστε το `gitlab.com` με το όνομα κεντρικού υπολογιστή της παρουσίας σας.

### Χρήση ξεχωριστού κλειδιού για το GitHub και άλλες υπηρεσίες

Εάν θέλετε ένα αποκλειστικό κλειδί για το GitHub (συνιστάται για την υγιεινή των κλειδιών), δημιουργήστε ένα κλειδί με όνομα:

```bash
ssh-keygen -t ed25519 -C "github" -f ~/.ssh/id_ed25519_github
```

Στη συνέχεια, προσθέστε ένα αποκλεισμό `Host` στο `~/.ssh/config`:

```
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_github
    AddKeysToAgent yes
```

Τώρα το `git clone git@github.com:...` χρησιμοποιεί αυτόματα το κλειδί για το GitHub.

## Δοκιμή και αντιμετώπιση προβλημάτων SSH

### Βασική δοκιμή συνδεσιμότητας

```bash
ssh -v user@server.example.com
```

Η σημαία `-v` (περιεκτική) δείχνει ακριβώς τι συμβαίνει σε κάθε βήμα. Αναζητήστε γραμμές όπως:
- `Offering public key: ~/.ssh/id_ed25519` — Το SSH δοκιμάζει το κλειδί σας
- `Server accepts key` — το κλειδί βρίσκεται στο `authorized_keys`
- `Authenticated to server.example.com` — επιτυχία

### Συνήθη σφάλματα και διορθώσεις

**"Δεν επιτρέπεται η άδεια (δημόσιο κλειδί)"**

Αυτό είναι το πιο συνηθισμένο σφάλμα. Αιτίες και διορθώσεις:

```bash
# Ελέγξτε ότι το κλειδί σας είναι φορτωμένο στο ssh-agent
ssh-add -l

# Εάν δεν αναφέρονται κλειδιά, προσθέστε το κλειδί σας
ssh-add ~/.ssh/id_ed25519

# Στον απομακρυσμένο διακομιστή, ελέγξτε τα authorized_keys
cat ~/.ssh/authorized_keys
# Βεβαιωθείτε ότι το δημόσιο κλειδί σας βρίσκεται εκεί

# Ελέγξτε τα δικαιώματα στον απομακρυσμένο διακομιστή
ls -la ~/.ssh/
# Το ~/.ssh πρέπει να είναι 700
# Τα authorized_keys πρέπει να είναι 600
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# Ελέγξτε ότι ο διακομιστής επιτρέπει την πιστοποίηση δημόσιου κλειδιού
sudo grep PubkeyAuthentication /etc/ssh/sshd_config
# Πρέπει να είναι: PubkeyAuthentication ναι
```

**"ΠΡΟΕΙΔΟΠΟΙΗΣΗ: Η ΑΝΑΓΝΩΡΙΣΗ ΑΠΟΣΤΑΣΕΙΣ HOST ΕΧΕΙ ΑΛΛΑΞΕΙ"**

Το κλειδί κεντρικού υπολογιστή του διακομιστή άλλαξε, το οποίο το SSH αντιμετωπίζει ως πιθανό ζήτημα ασφαλείας. Εάν γνωρίζετε ότι αυτό είναι νόμιμο (ο διακομιστής ανακατασκευάστηκε, η IP εκχωρήθηκε εκ νέου), αφαιρέστε την παλιά καταχώρηση:

```bash
# Καταργήστε τη συγκεκριμένη καταχώρηση κεντρικού υπολογιστή
ssh-keygen -R server.example.com

# Εναλλακτικά, επεξεργαστείτε το known_hosts με μη αυτόματο τρόπο και διαγράψτε τη γραμμή που επηρεάζεται
nano ~/.ssh/known_hosts
```

**"Εσφαλμένα δικαιώματα" ή "Τα δικαιώματα είναι πολύ ανοιχτά"**

Το SSH είναι αυστηρό σχετικά με τα δικαιώματα βασικών αρχείων:

```bash
# Διορθώστε τα δικαιώματα ιδιωτικού κλειδιού
chmod 600 ~/.ssh/id_ed25519

# Διορθώστε τα δικαιώματα δημόσιου κλειδιού
chmod 644 ~/.ssh/id_ed25519.pub

# Διορθώστε τα δικαιώματα καταλόγου .ssh
chmod 700 ~/.ssh
```

Στα Windows, τα δικαιώματα ιδιωτικού κλειδιού διαχειρίζονται διαφορετικά. Κάντε δεξί κλικ στο αρχείο κλειδιού → Ιδιότητες → Ασφάλεια → Για προχωρημένους → Απενεργοποίηση κληρονομικότητας → Κατάργηση όλων εκτός από τον λογαριασμό χρήστη σας, παραχωρώντας τον πλήρη έλεγχο μόνο στον εαυτό σας.

### Ρύθμιση και χρήση του ssh-agent

Εάν σας ζητείται επανειλημμένα η φράση πρόσβασης, το `ssh-agent` δεν εκτελείται ή το κλειδί σας δεν προστίθεται:

```bash
# Ξεκινήστε τον πράκτορα
eval "$(ssh-agent -s)"
# Πράκτορας pid 12345

# Προσθέστε το κλειδί σας
ssh-add ~/.ssh/id_ed25519
# Εισαγάγετε τη φράση πρόσβασης για /home/user/.ssh/id_ed25519:
# Προστέθηκε ταυτότητα: /home/user/.ssh/id_ed25519

# Επαληθεύω
ssh-add -l
# 256 SHA256:abc123... your@email.com (ED25519)
```

### Προσυμπλήρωση γνωστών_κεντρικών υπολογιστών στον Αυτοματισμό

Σε αγωγούς CI/CD, χρησιμοποιήστε το `ssh-keyscan` για να προσθέσετε το δακτυλικό αποτύπωμα του απομακρυσμένου κεντρικού υπολογιστή πριν από τη σύνδεση:

```bash
ssh-keyscan -H server.example.com >> ~/.ssh/known_hosts
ssh-keyscan -H github.com >> ~/.ssh/known_hosts
```

Αυτό αποτρέπει τις διαδραστικές προτροπές δακτυλικών αποτυπωμάτων σε αυτοματοποιημένα σενάρια.

## Συχνές Ερωτήσεις

### Πώς μπορώ να διορθώσω την "Απόρριψη άδειας (δημόσιο κλειδί)";

Επεξεργαστείτε αυτήν τη λίστα ελέγχου: (1) Βεβαιωθείτε ότι το κλειδί σας βρίσκεται στον πράκτορα SSH: `ssh-add -l`. Εάν όχι, εκτελέστε το `ssh-add ~/.ssh/id_ed25519`. (2) Ελέγξτε ότι το δημόσιο κλειδί σας βρίσκεται στο αρχείο `~/.ssh/authorized_keys` του διακομιστή. (3) Επαληθεύστε τα δικαιώματα: το `~/.ssh` πρέπει να είναι `700`, το `authorized_keys` πρέπει να είναι `600`. (4) Ελέγξτε ότι ο διακομιστής έχει `PubkeyAuthentication yes` στο `/etc/ssh/sshd_config`. (5) Χρησιμοποιήστε το `ssh -vvv user@server` για να δείτε ακριβώς πού αποτυγχάνει ο έλεγχος ταυτότητας. Η αναλυτική έξοδος θα σας πει ποια κλειδιά δοκιμάζονται και εάν ο διακομιστής τα αποδέχτηκε.

### Πώς μπορώ να προσθέσω το κλειδί SSH στο ssh-agent;

Εκτελέστε το `ssh-add ~/.ssh/id_ed25519` (αντικαταστήστε με το όνομα του αρχείου κλειδιού σας). Εάν ο πράκτορας δεν εκτελείται, ξεκινήστε τον πρώτα με το `eval "$(ssh-agent -s)"`. Στο macOS, χρησιμοποιήστε το `ssh-add --apple-use-keychain ~/.ssh/id_ed25519` για να αποθηκεύσετε τη φράση πρόσβασης στο Keychain. Στα Windows, ενεργοποιήστε την υπηρεσία `ssh-agent` με `Start-Service ssh-agent` και, στη συνέχεια, εκτελέστε το `ssh-add`. Χρειάζεται να εκτελείτε το `ssh-add` μόνο μία φορά ανά περίοδο λειτουργίας (ή μία φορά ποτέ, με το Keychain στο macOS).

### Μπορώ να έχω πολλά κλειδιά SSH;

Ναι, και είναι καλή πρακτική. Δημιουργήστε ξεχωριστά κλειδιά για διαφορετικούς σκοπούς — για παράδειγμα, ένα για προσωπικό GitHub, ένα για GitLab εργασίας, ένα για διακομιστές παραγωγής. Χρησιμοποιήστε το `~/.ssh/config` για να αντιστοιχίσετε κάθε κεντρικό υπολογιστή στο συγκεκριμένο κλειδί του με το `IdentityFile`. Η ύπαρξη χωριστών κλειδιών σημαίνει ότι η ανάκληση ενός (με την κατάργησή του από μια υπηρεσία ή διακομιστή) δεν επηρεάζει άλλα, περιορίζοντας την ακτίνα έκρηξης ενός κλειδιού σε κίνδυνο.

### Πώς μπορώ να αλλάξω τη φράση πρόσβασης σε ένα κλειδί SSH;

Χρησιμοποιήστε το `ssh-keygen -p`:

```bash
ssh-keygen -p -f ~/.ssh/id_ed25519
```

Θα σας ζητηθεί η τρέχουσα φράση πρόσβασης και, στη συνέχεια, θα σας ζητηθεί να εισαγάγετε και να επιβεβαιώσετε τη νέα. Το ίδιο το κλειδί (και επομένως το δημόσιο κλειδί) δεν αλλάζει — δεν χρειάζεται να το ενημερώσετε πουθενά. Αυτό είναι χρήσιμο εάν υποψιάζεστε ότι η φράση πρόσβασής σας έχει παραβιαστεί ή θέλετε να την εναλλάξετε ως συνήθη πρακτική ασφαλείας.

### Πώς μπορώ να αφαιρέσω ένα παλιό κλειδί SSH από έναν διακομιστή;

Ανοίξτε το `~/.ssh/authorized_keys` στον διακομιστή και διαγράψτε τη γραμμή που περιέχει το παλιό δημόσιο κλειδί:

```bash
# Επεξεργαστείτε τα authorized_keys στον απομακρυσμένο διακομιστή
ssh user@server.example.com
nano ~/.ssh/authorized_keys
# Διαγράψτε τη γραμμή για το κλειδί που θέλετε να αφαιρέσετε
# Αποθήκευση και έξοδος
```

Κάθε γραμμή στο `authorized_keys` είναι ένα δημόσιο κλειδί. Το σχόλιο του κλειδιού (το τελευταίο πεδίο, συνήθως `user@hostname` ή ένα email) σας βοηθά να προσδιορίσετε ποιο κλειδί είναι ποιο. Εάν δεν γνωρίζετε ποια γραμμή να διαγράψετε, συγκρίνετε το δακτυλικό αποτύπωμα:

```bash
# Στο τοπικό σας μηχάνημα, λάβετε το δακτυλικό αποτύπωμα
ssh-keygen -lf ~/.ssh/id_ed25519.pub

# Στον διακομιστή, ελέγξτε τα δακτυλικά αποτυπώματα όλων των εξουσιοδοτημένων κλειδιών
ssh-keygen -lf ~/.ssh/authorized_keys
```

Αντιστοιχίστε τα δακτυλικά αποτυπώματα για να αναγνωρίσετε και να διαγράψετε το σωστό κλειδί.

## Δημιουργήστε τα κλειδιά SSH σας δωρεάν

Εάν δεν έχετε δημιουργήσει ακόμα το ζεύγος κλειδιών SSH, η δωρεάν διαδικτυακή μας γεννήτρια δημιουργεί κλειδιά Ed25519 ή RSA απευθείας στο πρόγραμμα περιήγησής σας σε δευτερόλεπτα. Δεν απαιτείται εγκατάσταση, τίποτα δεν αποστέλλεται στους διακομιστές μας — η παραγωγή γίνεται εξ ολοκλήρου στη συσκευή σας.
