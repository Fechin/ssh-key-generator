# Η εντολή SSH: Ολοκληρωμένη αναφορά με παραδείγματα για Linux, Mac και Windows

Η εντολή `ssh` είναι το κύριο εργαλείο για την έναρξη συνδέσεων SSH από τη γραμμή εντολών. Είτε συνδέεστε σε έναν απομακρυσμένο διακομιστή Linux, είτε προωθείτε μια θύρα για πρόσβαση στη βάση δεδομένων είτε πηδάτε μέσα από έναν κεντρικό υπολογιστή προμαχώνα για να φτάσετε σε ένα εσωτερικό δίκτυο, η εντολή `ssh` είναι το σημείο εισόδου σας. Αυτός ο οδηγός καλύπτει τα πάντα, από τη βασική σύνταξη έως τις προηγμένες σημαίες, τη δημιουργία σήραγγας και τις διαμορφώσεις ProxyJump — με πραγματικά παραδείγματα για Linux, macOS και Windows.

Πριν εκτελέσετε εντολές SSH, θα χρειαστείτε κλειδιά SSH. Δείτε τον οδηγό μας σχετικά με το <a href="/el/what-is-an-ssh-key/">τι είναι ένα κλειδί SSH</a> ή χρησιμοποιήστε τη <a href="/">δωρεάν ηλεκτρονική γεννήτρια</a> για να δημιουργήσετε ένα ζεύγος κλειδιών.

## Σύνταξη εντολών SSH

Η βασική σύνταξη της εντολής `ssh` είναι:

```
ssh [options] [user@]hostname [command]
```

Αναλύοντας αυτό:

- **`[options]`** — Μηδέν ή περισσότερες σημαίες που τροποποιούν τη συμπεριφορά (καλύπτονται σε αυτόν τον οδηγό)
- **`[user@]`** — Προαιρετικό όνομα χρήστη. Εάν παραλειφθεί, το SSH χρησιμοποιεί το τρέχον τοπικό σας όνομα χρήστη
- **`hostname`** — Ο απομακρυσμένος κεντρικός υπολογιστής στον οποίο πρέπει να συνδεθείτε. Μπορεί να είναι διεύθυνση IP, όνομα τομέα ή ψευδώνυμο που ορίζεται στο `~/.ssh/config`
- **`[command]`** — Προαιρετική εντολή για εκτέλεση στον απομακρυσμένο κεντρικό υπολογιστή. Εάν παραλειφθεί, το SSH ξεκινά μια διαδραστική περίοδο λειτουργίας φλοιού

**Βασικά παραδείγματα:**

```bash
# Συνδεθείτε χρησιμοποιώντας το τρέχον όνομα χρήστη σας
ssh server.example.com

# Συνδεθείτε ως συγκεκριμένος χρήστης
ssh alice@server.example.com

# Συνδεθείτε χρησιμοποιώντας μια διεύθυνση IP
ssh ubuntu@192.168.1.100

# Εκτελέστε μία μόνο εντολή και αποσυνδέστε αμέσως
ssh user@server.example.com "df -h"

# Εκτελέστε μια εντολή και μείνετε συνδεδεμένοι
ssh user@server.example.com
```

## SSH σε Linux και macOS

Οι διανομές Linux και το macOS αποστέλλονται με προεγκατεστημένο το **OpenSSH**. Στο Linux, είναι μέρος του πακέτου `openssh-client` (Debian/Ubuntu) ή `openssh-clients` (RHEL/CentOS/Fedora). Στο macOS, συνοδεύεται από το σύστημα και είναι άμεσα διαθέσιμο από το Terminal ή το iTerm2.

### Επαληθεύστε ότι το SSH είναι διαθέσιμο

```bash
ssh -V
# Έξοδος: OpenSSH_9.6p1, LibreSSL 3.3.6
```

### Βασική σύνδεση

```bash
# Συνδεθείτε σε διακομιστή
ssh user@myserver.example.com

# Το SSH ελέγχει το ~/.ssh/known_hosts στην πρώτη σύνδεση και ζητά:
# Η αυθεντικότητα του κεντρικού υπολογιστή "myserver.example.com" δεν μπορεί να εξακριβωθεί.
# Το δακτυλικό αποτύπωμα του κλειδιού ED25519 είναι SHA256:abc123...
# Είστε βέβαιοι ότι θέλετε να συνεχίσετε τη σύνδεση (ναι/όχι); Ναί

# Το δακτυλικό αποτύπωμα του κεντρικού υπολογιστή αποθηκεύεται στο ~/.ssh/known_hosts
```

### Χρήση αρχείου διαμόρφωσης SSH για ευκολία

Αντί να πληκτρολογείτε πλήρεις λεπτομέρειες σύνδεσης κάθε φορά, δημιουργήστε `~/.ssh/config`:

```
Host myserver
    HostName myserver.example.com
    User alice
    Port 22
    IdentityFile ~/.ssh/id_ed25519

Host dev
    HostName dev.example.com
    User ubuntu
    Port 2222
    IdentityFile ~/.ssh/id_ed25519_work
```

Τώρα το `ssh myserver` επεκτείνεται στην πλήρη σύνδεση — δεν χρειάζονται σημαίες.

## SSH σε Windows (PowerShell και OpenSSH)

Τα Windows 10 (έκδοση 1809) και τα Windows 11 περιλαμβάνουν το **OpenSSH-πελάτη** ως προαιρετική δυνατότητα, καθιστώντας την εντολή `ssh` διαθέσιμη στο PowerShell, στη γραμμή εντολών και στο Windows Terminal.

### Ελέγξτε εάν το OpenSSH είναι εγκατεστημένο

```powershell
# Στο PowerShell
ssh -V
```

Εάν η εντολή δεν βρεθεί, εγκαταστήστε την:

**Μέσω ρυθμίσεων:**
1. Ανοίξτε **Ρυθμίσεις → Εφαρμογές → Προαιρετικές λειτουργίες**
2. Κάντε κλικ στο **Προσθήκη δυνατότητας**
3. Βρείτε το **OpenSSH Client** και εγκαταστήστε το

**Μέσω PowerShell (Διαχειριστής):**

```powershell
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0
```

### Χρήση SSH στο PowerShell

Μόλις εγκατασταθεί, η σύνταξη είναι πανομοιότυπη με το Linux και το macOS:

```powershell
# Συνδεθείτε σε διακομιστή
ssh user@server.example.com

# Συνδεθείτε σε μια προσαρμοσμένη θύρα
ssh -p 2222 user@server.example.com

# Χρησιμοποιήστε ένα συγκεκριμένο αρχείο κλειδιού
ssh -i C:\Users\YourName\.ssh\id_ed25519 user@server.example.com
```

### Θέση κλειδιού SSH των Windows

Τα κλειδιά SSH στα Windows αποθηκεύονται στο `C:\Users\YourUsername\.ssh\`. Το PowerShell αναγνωρίζει το `~` ως συντομογραφία:

```powershell
# Αυτά είναι ισοδύναμα
ssh -i ~/.ssh/id_ed25519 user@server
ssh -i C:\Users\YourUsername\.ssh\id_ed25519 user@server
```

### PuTTY εναντίον OpenSSH στα Windows

Πριν το OpenSSH ήταν διαθέσιμο στα Windows, το **PuTTY** ήταν ο τυπικός πελάτης SSH. Το PuTTY χρησιμοποιεί τη δική του μορφή κλειδιού (αρχεία `.ppk`) και τη δική του γεννήτρια κλειδιών (PuTTYgen). Εάν έχετε κλειδί `.ppk` και πρέπει να το μετατρέψετε στην τυπική μορφή OpenSSH:

```bash
# Χρήση εργαλείων PuTTY
puttygen mykey.ppk -O private-openssh -o id_ed25519
```

Για νέες ρυθμίσεις, προτιμήστε το ενσωματωμένο πρόγραμμα-πελάτη OpenSSH στα Windows — διατηρείται από το έργο OpenSSH και χρησιμοποιεί την ίδια μορφή κλειδιού και αρχείο διαμόρφωσης όπως το Linux και το macOS.

## SSH με έλεγχο ταυτότητας κλειδιού: Η σημαία -i

Από προεπιλογή, το SSH αναζητά αρχεία κλειδιών στο `~/.ssh/` με τυπικά ονόματα (`id_ed25519`, `id_rsa`, `id_ecdsa`, `id_dsa`). Η σημαία **`-i`** (αρχείο ταυτότητας) σάς επιτρέπει να καθορίσετε ρητά ένα διαφορετικό αρχείο κλειδιού.

```bash
# Χρησιμοποιήστε ένα συγκεκριμένο αρχείο κλειδιού
ssh -i ~/.ssh/id_ed25519_production user@prod.example.com

# Χρησιμοποιήστε ένα κλειδί που είναι αποθηκευμένο σε μη τυπική τοποθεσία
ssh -i /path/to/my/custom_key deploy@server.example.com

# Συνδυάστε με άλλες επιλογές
ssh -i ~/.ssh/deploy_key -p 2222 -l deploy server.example.com
```

Η σημαία `-i` είναι απαραίτητη σε πολλές περιπτώσεις:
- Έχετε πολλά ζεύγη κλειδιών για διαφορετικούς διακομιστές ή σκοπούς
- Το κλειδί σας δεν βρίσκεται στον προεπιλεγμένο κατάλογο `~/.ssh/`
- Καθορίζετε ένα κλειδί σε περιβάλλον CI/CD όπου η διαδρομή είναι σαφής
- Κατεβάσατε ένα κλειδί `.pem` από το AWS ή άλλο πάροχο cloud

### Παράδειγμα AWS EC2

Το AWS παρέχει ένα αρχείο κλειδιού `.pem` όταν εκκινείτε μια παρουσία EC2:

```bash
# Συνδεθείτε σε μια παρουσία AWS EC2 με ένα κλειδί .pem
ssh -i ~/Downloads/my-aws-key.pem ec2-user@54.123.456.789

# Διορθώστε τα δικαιώματα εάν χρειάζεται (απαιτείται — το SSH απορρίπτει κλειδιά αναγνώσιμα από τον κόσμο)
chmod 400 ~/Downloads/my-aws-key.pem
```

## Βασικές επιλογές και σημαίες εντολών SSH

Η εντολή `ssh` έχει δεκάδες επιλογές. Εδώ είναι αυτά που θα χρησιμοποιήσετε πραγματικά.

| Σημαία | Μεγάλη φόρμα / Περιγραφή | Παράδειγμα |
|---|---|---|
| `-p PORT` | Σύνδεση σε μη τυπική θύρα | `ssh -p 2222 user@host` |
| `-i FILE` | Χρησιμοποιήστε μια συγκεκριμένη ταυτότητα (κλειδί) αρχείο | `ssh -i ~/.ssh/mykey user@host` |
| `-l USER` | Καθορίστε το όνομα χρήστη σύνδεσης (εναλλακτικό στο `user@`) | `ssh -l alice host` |
| `-L local:host:remote` | Τοπική προώθηση λιμένων | `ssh -L 8080:localhost:80 host` |
| `-R remote:host:local` | Απομακρυσμένη προώθηση θύρας | `ssh -R 9090:localhost:3000 host` |
| `-D port` | Δυναμικός (SOCKS) διακομιστής μεσολάβησης | `ssh -D 1080 host` |
| `-N` | Μην εκτελείτε απομακρυσμένη εντολή (μόνο διοχέτευση σήραγγας) | `ssh -N -L 3307:db:3306 host` |
| `-f` | Μετάβαση στο παρασκήνιο μετά τον έλεγχο ταυτότητας | `ssh -f -N -D 1080 host` |
| `-v` | Λεπτομερής έξοδος (debug) | `ssh -v user@host` |
| `-vv` ή `-vvv` | Πιο αναλυτικό αποτέλεσμα | `ssh -vvv user@host` |
| `-A` | Ενεργοποίηση προώθησης πράκτορα SSH | `ssh -A user@jump-host` |
| `-X` | Ενεργοποίηση προώθησης X11 | `ssh -X user@host xclock` |
| `-C` | Ενεργοποίηση συμπίεσης | `ssh -C user@host` |
| `-T` | Απενεργοποίηση της κατανομής ψευδοτερματικού | `ssh -T git@github.com` |
| `-q` | Αθόρυβη λειτουργία (καταστολή προειδοποιήσεων) | `ssh -q user@host` |
| `-o OPTION=VALUE` | Ορίστε μια επιλογή διαμόρφωσης inline | `ssh -o StrictHostKeyChecking=no host` |
| `-J JUMP` | ProxyJump μέσω ενός προμαχώνα υποδοχής | `ssh -J jump.host user@internal` |
| `-W host:port` | Προώθηση του stdio σε host:port (για ProxyJump) | Χρησιμοποιείται εσωτερικά από `-J` |

### Λεπτή λειτουργία για εντοπισμό σφαλμάτων

Όταν το SSH αποτυγχάνει, το `-v` (ή `-vv` ή `-vvv`) αποκαλύπτει ακριβώς τι συμβαίνει:

```bash
ssh -v user@server.example.com
# Εμφανίζει: ποια κλειδιά δοκιμάζονται, ποιες μεθόδους εξουσιοδότησης αποδέχεται ο διακομιστής,
# εάν εμπλέκεται ο πράκτορας και πού αποτυγχάνει η σύνδεση

ssh -vvv user@server.example.com
# Μέγιστη πολυγλωσσία — δείχνει λεπτομερή κρυπτογραφική διαπραγμάτευση
```

### Παράκαμψη ελέγχου κλειδιού κεντρικού υπολογιστή (Χρησιμοποιήστε με προσοχή)

Σε περιβάλλοντα αυτοματισμού και δοκιμών όπου η επαλήθευση κλειδιού κεντρικού υπολογιστή δεν είναι βολική:

```bash
ssh -o StrictHostKeyChecking=no -o UserKnownHostsFile=/dev/null user@host
```

**Μην το χρησιμοποιείτε στην παραγωγή.** Απενεργοποιεί την προστασία από επιθέσεις man-in-the-middle. Αντίθετα, συμπληρώστε εκ των προτέρων το `known_hosts` χρησιμοποιώντας το `ssh-keyscan` στα σενάρια παροχής.

### Agent Forwarding

```bash
# Προωθήστε τον τοπικό σας πράκτορα SSH στον απομακρυσμένο διακομιστή
ssh -A user@jump-server

# Από το jump-server, μπορείτε πλέον να SSH σε εσωτερικούς κεντρικούς υπολογιστές χρησιμοποιώντας τα τοπικά σας κλειδιά
# Δεν χρειάζεται να αντιγράψετε ιδιωτικά κλειδιά στον διακομιστή jump
ssh internal-server.private
```

Η προώθηση πράκτορα είναι χρήσιμη, αλλά έχει επιπτώσεις στην ασφάλεια: οποιοσδήποτε έχει πρόσβαση root στον ενδιάμεσο διακομιστή μπορεί να χρησιμοποιήσει τον προωθημένο πράκτορά σας. Χρησιμοποιήστε το μόνο με αξιόπιστους διακομιστές και προτιμήστε το **ProxyJump** όταν είναι δυνατόν (δεν εκθέτει τον πράκτορά σας σε ενδιάμεσους κεντρικούς υπολογιστές).

## Εντολές SSH Tunneling

Η σήραγγα SSH δρομολογεί συνδέσεις TCP μέσω ενός κρυπτογραφημένου καναλιού SSH. Για πληροφορίες σχετικά με το τι είναι η σήραγγα, ανατρέξτε στην <a href="/el/what-is-ssh/">ενότητα τούνελ του οδηγού SSH</a>.

### Τοπική προώθηση λιμένων

Κάνει μια απομακρυσμένη υπηρεσία προσβάσιμη στον τοπικό σας υπολογιστή.

```bash
# Πρόσβαση σε μια απομακρυσμένη βάση δεδομένων MySQL στην τοπική θύρα 3307
ssh -L 3307:localhost:3306 user@db-server.example.com -N

# Συνδεθείτε σε έναν απομακρυσμένο διακομιστή web που είναι προσβάσιμος μόνο από τον κεντρικό υπολογιστή SSH
ssh -L 8080:intranet.company.com:80 user@vpn.company.com -N

# Πρόσβαση σε μια παρουσία Redis σε ένα ιδιωτικό δίκτυο
ssh -L 6380:10.0.1.50:6379 user@bastion.example.com -N
```

Αφού εκτελέσετε οποιοδήποτε από τα παραπάνω, συνδέστε την τοπική σας εφαρμογή στο `127.0.0.1:LOCAL_PORT`.

### Απομακρυσμένη προώθηση θύρας

Εκθέτει μια τοπική υπηρεσία στον απομακρυσμένο διακομιστή.

```bash
# Εκθέστε τον τοπικό διακομιστή ανάπτυξης (θύρα 3000) στη θύρα 9090 του απομακρυσμένου διακομιστή
ssh -R 9090:localhost:3000 user@remote.example.com -N

# Να επιτρέπεται σε οποιαδήποτε διεπαφή στον απομακρυσμένο διακομιστή να συνδεθεί (απαιτούνται GatewayPorts ναι στο sshd_config)
ssh -R 0.0.0.0:9090:localhost:3000 user@remote.example.com -N
```

### Dynamic Port Forwarding (SOCKS Proxy)

```bash
# Δημιουργήστε έναν διακομιστή μεσολάβησης SOCKS5 στην τοπική θύρα 1080, που εκτελείται στο παρασκήνιο
ssh -D 1080 -N -f user@remote.example.com

# Τώρα διαμορφώστε το πρόγραμμα περιήγησης ή το σύστημά σας ώστε να χρησιμοποιεί το διακομιστή μεσολάβησης SOCKS5: 127.0.0.1:1080
# Όλη η κίνηση δρομολογείται μέσω του απομακρυσμένου διακομιστή
```

Για χρήση με μπούκλα:

```bash
curl --socks5 127.0.0.1:1080 https://api.example.com/
```

### Εκτέλεση του τούνελ στο παρασκήνιο

```bash
# -f στέλνει SSH στο παρασκήνιο, -N σημαίνει καμία απομακρυσμένη εντολή
ssh -f -N -L 3307:localhost:3306 user@db-server

# Βρείτε τη διαδικασία SSH στο παρασκήνιο
ps aux | grep ssh

# Σκότωσε το
kill PID
```

## SSH ProxyJump και Bastion Host

Σε πολλούς οργανισμούς, οι διακομιστές παραγωγής βρίσκονται σε ιδιωτικά δίκτυα που δεν είναι άμεσα προσβάσιμοι από το Διαδίκτυο. Ένας **προπύργιος κεντρικός υπολογιστής** (ή κεντρικός υπολογιστής μεταπήδησης) είναι ένας σκληρυμένος διακομιστής στην περίμετρο του δικτύου στον οποίο κάνετε SSH πρώτα και μετά μεταπηδάτε σε εσωτερικούς διακομιστές. Η σημαία `ssh -J` (ProxyJump, προστέθηκε στο OpenSSH 7.3) το καθιστά απρόσκοπτο.

### Βασικό ProxyJump

```bash
# Περάστε από έναν προμαχώνα για να φτάσετε σε έναν εσωτερικό διακομιστή
ssh -J ec2-user@bastion.example.com ubuntu@10.0.1.50

# Σύντομη φόρμα
ssh -J user@jump-host user@internal-server
```

Αυτό δημιουργεί δύο συνδέσεις SSH: πρώτα στον προμαχώνα, μετά από τον προμαχώνα στον εσωτερικό διακομιστή — όλα σε μία εντολή, με κρυπτογράφηση από άκρο σε άκρο. Ο τοπικός σας πράκτορας SSH χειρίζεται τον έλεγχο ταυτότητας και στους δύο κεντρικούς υπολογιστές.

### Αλυσίδα πολλαπλών αλμάτων

```bash
# Μετάβαση σε δύο ενδιάμεσους οικοδεσπότες
ssh -J user@jump1.example.com,user@jump2.example.com user@final-destination
```

### ProxyJump στο αρχείο διαμόρφωσης SSH

Για διακομιστές με συχνή πρόσβαση, βάλτε τη διαμόρφωση ProxyJump στο `~/.ssh/config`:

```
Host bastion
    HostName bastion.example.com
    User ec2-user
    IdentityFile ~/.ssh/id_ed25519_aws

Host prod-web
    HostName 10.0.1.100
    User ubuntu
    ProxyJump bastion
    IdentityFile ~/.ssh/id_ed25519_aws

Host prod-db
    HostName 10.0.2.50
    User postgres
    ProxyJump bastion
    IdentityFile ~/.ssh/id_ed25519_aws
```

Τώρα το `ssh prod-web` συνδέεται αυτόματα μέσω του προμαχώνα. Δεν χρειάζονται χειροκίνητες σημαίες `-J`.

### ProxyJump vs Agent Forwarding

Το ProxyJump προτιμάται έναντι της προώθησης πράκτορα (`-A`) επειδή:
- Το ιδιωτικό κλειδί σας δεν προωθείται ποτέ στον ενδιάμεσο κεντρικό υπολογιστή
- Ο οικοδεσπότης του προμαχώνα δεν μπορεί να χρησιμοποιήσει τα διαπιστευτήριά σας για έλεγχο ταυτότητας αλλού
- Η σύνδεση επαληθεύεται από άκρο σε άκρο — ο διακομιστής προορισμού σάς επαληθεύει απευθείας

Για περισσότερες λεπτομέρειες ρύθμισης, ανατρέξτε στον πλήρη οδηγό μας σχετικά με τον <a href="/el/how-to-set-up-ssh/">πώς να ρυθμίσετε το SSH</a>.

## Συχνές Ερωτήσεις

### Πώς μπορώ να εκτελέσω μια εντολή SSH χωρίς κωδικό πρόσβασης;

Ρυθμίστε τον έλεγχο ταυτότητας κλειδιού SSH. Δημιουργήστε ένα ζεύγος κλειδιών (`ssh-keygen -t ed25519`), αντιγράψτε το δημόσιο κλειδί στο αρχείο `~/.ssh/authorized_keys` του διακομιστή χρησιμοποιώντας το `ssh-copy-id user@host` και οι επόμενες συνδέσεις θα χρησιμοποιήσουν το κλειδί σας χωρίς να σας ζητηθεί κωδικός πρόσβασης. Εάν το κλειδί σας έχει φράση πρόσβασης, χρησιμοποιήστε τα `ssh-agent` και `ssh-add` για να το αποθηκεύσετε προσωρινά, ώστε να το εισάγετε μόνο μία φορά ανά περίοδο λειτουργίας.

### Πώς βγαίνω από μια περίοδο λειτουργίας SSH;

Πληκτρολογήστε `exit` ή πατήστε **Ctrl+D** για να κλείσετε το απομακρυσμένο κέλυφος και να αποσυνδεθείτε. Εάν η περίοδος λειτουργίας έχει παγώσει (π.χ. πέσει το δίκτυο), χρησιμοποιήστε την ακολουθία διαφυγής SSH: πατήστε **Enter** και, στη συνέχεια, πληκτρολογήστε `~.` (tilde ακολουθούμενη από τελεία). Το SSH τερματίζει αμέσως τη σύνδεση. Άλλες ακολουθίες διαφυγής: το `~^Z` αναστέλλει το SSH στο παρασκήνιο, το `~?` παραθέτει όλες τις ακολουθίες διαφυγής.

### Πώς μπορώ να εκτελέσω μια εντολή σε έναν απομακρυσμένο διακομιστή μέσω SSH;

Προσθέστε την εντολή μετά το όνομα κεντρικού υπολογιστή:

```bash
ssh user@server "command to run"
ssh user@server "ls -la /var/log/"
ssh user@server "sudo systemctl status nginx"
```

Για εντολές ή σενάρια πολλαπλών γραμμών, χρησιμοποιήστε ένα heredoc:

```bash
ssh user@server << 'EOF'
cd /var/www/html
git pull origin main
systemctl reload nginx
EOF
```

Για διαδραστικές εντολές (όπως αυτές που απαιτούν τερματικό), προσθέστε τη σημαία `-t` για να εκχωρήσετε ένα ψευδοτερματικό: `ssh -t user@server sudo apt upgrade`.

### Ποια είναι η διαφορά μεταξύ ssh και scp;

Το `ssh` ανοίγει μια περίοδο λειτουργίας τερματικού (ή εκτελεί μια μεμονωμένη εντολή) σε ένα απομακρυσμένο μηχάνημα. Το `scp` (Πρωτόκολλο ασφαλούς αντιγραφής) αντιγράφει αρχεία μεταξύ του τοπικού σας υπολογιστή και ενός απομακρυσμένου υπολογιστή ή μεταξύ δύο απομακρυσμένων μηχανημάτων, χρησιμοποιώντας SSH για κρυπτογράφηση και έλεγχο ταυτότητας. Και οι δύο χρησιμοποιούν το ίδιο πρωτόκολλο SSH και την ίδια υποδομή κλειδιών. Για μεταφορές αρχείων, το `scp` είναι το αποκλειστικό εργαλείο:

```bash
# scp από τοπικό σε απομακρυσμένο
scp file.txt user@server:/destination/path/

# scp απομακρυσμένο σε τοπικό
scp user@server:/remote/file.txt ./local-copy.txt
```

Για πιο προηγμένες μεταφορές αρχείων (αναδρομικές, επαναλαμβανόμενες, περιήγηση καταλόγου), χρησιμοποιήστε το `sftp`. Για μεταφορές μεγάλης κλίμακας, το `rsync -avz -e ssh` είναι ακόμα πιο αποτελεσματικό.

### Πώς μπορώ να χρησιμοποιήσω το SSH με ένα αρχείο διαμόρφωσης;

Δημιουργήστε `~/.ssh/config` και προσθέστε καταχωρήσεις κεντρικού υπολογιστή. Κάθε μπλοκ `Host` ορίζει ένα ψευδώνυμο σύνδεσης με τις πλήρεις ρυθμίσεις του:

```
Host alias-name
    HostName actual.server.com
    User username
    Port 22
    IdentityFile ~/.ssh/specific_key
```

Στη συνέχεια, το `ssh alias-name` χρησιμοποιεί αυτόματα όλες τις καθορισμένες ρυθμίσεις. Το αρχείο διαμόρφωσης υποστηρίζει χαρακτήρες μπαλαντέρ: Το `Host *.example.com` εφαρμόζει ρυθμίσεις σε όλους τους αντίστοιχους κεντρικούς υπολογιστές. Το `Host *` στο τέλος εφαρμόζει προεπιλογές σε όλες τις συνδέσεις. Οι επιλογές σε ένα συγκεκριμένο μπλοκ `Host` παρακάμπτουν τις καθολικές προεπιλογές.

## Χρειάζεστε κλειδιά SSH; Δημιουργήστε τα δωρεάν

Για να χρησιμοποιήσετε τον έλεγχο ταυτότητας κλειδιού SSH — που εξαλείφει την ανάγκη για κωδικούς πρόσβασης — χρειάζεστε ένα ζεύγος κλειδιών. Δημιουργήστε κλειδιά Ed25519 ή RSA άμεσα στο πρόγραμμα περιήγησής σας, χωρίς να αποστέλλεται τίποτα στους διακομιστές μας.
