# Vygenerovat SSH klíč: online nástroj a příkazy ssh-keygen

Vygenerovat SSH klíč znamená vytvořit pár veřejného a soukromého klíče. Pro nové klíče je obvykle nejlepší Ed25519; RSA 4096 používejte kvůli starším systémům.

Online generátor vytvoří pár klíčů v prohlížeči. Veřejný klíč přidáte do GitHubu, GitLabu, Bitbucketu nebo na server; soukromý klíč zůstane na vašem počítači.

## Rychlá odpověď

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

<a href="/cs/">Generátor SSH klíčů: vytvořte SSH klíče online</a> Generujte Ed25519 nebo RSA SSH klíče v prohlížeči pro GitHub, GitLab, servery a ssh-keygen. Soukromý klíč se neodesílá.

## Vygenerovat SSH klíč online

1. Otevřete generátor SSH klíčů.
2. Zvolte Ed25519 pro moderní systémy nebo RSA 4096 pro starší kompatibilitu.
3. Přidejte jasný komentář, například e-mail nebo název projektu.
4. U osobních klíčů použijte passphrase.
5. Vygenerujte pár klíčů a soubory zkopírujte nebo stáhněte.
6. Na GitHub, GitLab, Bitbucket nebo server přidejte pouze veřejný klíč.

Veřejný klíč můžete sdílet. Soukromý klíč musí zůstat tajný a nepatří do ticketů, chatu ani dokumentace.

## Vygenerovat SSH klíč pomocí ssh-keygen

`ssh-keygen` / OpenSSH / Linux / macOS / Windows

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
```

## Přidat klíč do GitHubu, GitLabu nebo na server

Běžný postup: vygenerovat klíč, zkopírovat veřejný klíč, přidat ho do GitHubu/GitLabu nebo na server, načíst soukromý klíč do ssh-agent a otestovat spojení.

```bash
ssh -T git@github.com
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com
```

## Volba Ed25519 nebo RSA

| Použití | Doporučený klíč |
|---|---|
| GitHub, GitLab, Bitbucket | Ed25519 |
| Moderní Linux server | Ed25519 |
| CI/CD deploy klíč | Ed25519 |
| Starší SSH server | RSA 4096 |

Pro nové klíče většinou zvolte Ed25519. RSA 4096 použijte, pokud starší SSH server nebo nástroj nepodporuje Ed25519.

## Passphrase a ssh-agent

Veřejný klíč můžete sdílet. Soukromý klíč musí zůstat tajný a nepatří do ticketů, chatu ani dokumentace.

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## Časté problémy

Většině připojení stačí ssh user@host. Použijte -i pro konkrétní klíč, -p pro jiný port a -J pro připojení přes bastion.

```bash
ssh -v user@server.example.com
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
ssh -i ~/.ssh/id_ed25519_work user@server.example.com
```
