# إنشاء مفتاح SSH: مولّد عبر الإنترنت وأوامر ssh-keygen

إنشاء مفتاح SSH يعني إنشاء زوج من مفتاح عام ومفتاح خاص. للمفاتيح الجديدة يكون Ed25519 هو الخيار العملي غالبًا؛ استخدم RSA 4096 عند الحاجة للتوافق مع أنظمة قديمة.

ينشئ مولّد مفاتيح SSH عبر الإنترنت زوج المفاتيح داخل المتصفح. يُضاف المفتاح العام إلى GitHub أو GitLab أو Bitbucket أو الخادم؛ ويبقى المفتاح الخاص على جهازك.

## إجابة سريعة

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

<a href="/ar/">مولّد مفاتيح SSH: أنشئ مفاتيح SSH عبر الإنترنت</a> أنشئ مفاتيح SSH من نوع Ed25519 أو RSA داخل المتصفح لاستخدامها مع GitHub وGitLab والخوادم وssh-keygen. لا يتم رفع المفتاح الخاص.

## إنشاء مفتاح SSH عبر الإنترنت

1. افتح مولّد مفاتيح SSH.
2. اختر Ed25519 للأنظمة الحديثة، أو RSA 4096 للتوافق مع الأنظمة القديمة.
3. أضف تعليقًا واضحًا مثل البريد الإلكتروني أو اسم المشروع.
4. استخدم عبارة مرور للمفاتيح الشخصية.
5. أنشئ زوج المفاتيح ثم انسخ الملفات أو نزّلها.
6. ثبّت المفتاح العام فقط على GitHub أو GitLab أو Bitbucket أو الخادم.

يمكن مشاركة المفتاح العام. يجب أن يبقى المفتاح الخاص سريًا ولا يُلصق في التذاكر أو المحادثات أو الوثائق.

## إنشاء مفتاح SSH باستخدام ssh-keygen

`ssh-keygen` / OpenSSH / Linux / macOS / Windows

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
```

## إضافة المفتاح إلى GitHub أو GitLab أو خادم

المسار المعتاد: أنشئ المفتاح، انسخ المفتاح العام، أضفه إلى GitHub/GitLab أو الخادم، حمّل المفتاح الخاص في ssh-agent ثم اختبر الاتصال.

```bash
ssh -T git@github.com
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com
```

## اختيار Ed25519 أو RSA

| الاستخدام | المفتاح الموصى به |
|---|---|
| GitHub وGitLab وBitbucket | Ed25519 |
| خادم Linux حديث | Ed25519 |
| مفتاح نشر CI/CD | Ed25519 |
| خادم SSH قديم | RSA 4096 |

للمفاتيح الجديدة اختر Ed25519 غالبًا. استخدم RSA 4096 إذا كان خادم SSH قديمًا أو أداة قديمة لا تدعم Ed25519.

## عبارة المرور وssh-agent

يمكن مشاركة المفتاح العام. يجب أن يبقى المفتاح الخاص سريًا ولا يُلصق في التذاكر أو المحادثات أو الوثائق.

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## مشكلات شائعة

في معظم الاتصالات يكفي ssh user@host. استخدم -i لاختيار مفتاح محدد، و-p لمنفذ مختلف، و-J للمرور عبر خادم وسيط.

```bash
ssh -v user@server.example.com
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
ssh -i ~/.ssh/id_ed25519_work user@server.example.com
```
