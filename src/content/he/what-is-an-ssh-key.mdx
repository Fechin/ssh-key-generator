# מה זה מפתח SSH? מפתחות ציבוריים, פרטיים וזוגות מפתחות — הסבר

**מפתח SSH** הוא פרטי גישה קריפטוגרפיים המשמשים לאימות בשרתי SSH ללא סיסמה. מפתחות SSH מגיעים בזוגות — **מפתח ציבורי** ו**מפתח פרטי** — ויחד הם הופכים את הגישה מרחוק לבטוחה ונוחה יותר מאימות סיסמה מסורתי. המדריך הזה מסביר מה זה מפתח SSH, איך מערכת זוג המפתחות עובדת, איזה סוג מפתח לבחור, וכיצד להשתמש במפתחות SSH עם שירותים כמו GitHub ו-GitLab.

## איך מפתחות SSH עובדים

מפתחות SSH מבוססים על **קריפטוגרפיה אסימטרית**, הנקראת גם קריפטוגרפיה של מפתח ציבורי. בניגוד לקריפטוגרפיה סימטרית (שבה מפתח סוד יחיד גם מצפין וגם מפענח נתונים), מערכות אסימטריות משתמשות בזוגות מפתחות הקשורים מתמטית שבהם נתונים המוצפנים עם מפתח אחד ניתן לפענוח רק עם השני.

זהו זרימת האימות כשאתה משתמש במפתח SSH להתחבר לשרת:

1. אתה מריץ `ssh user@server.example.com`.
2. השרת מחפש את המפתח הציבורי שלך ב-`~/.ssh/authorized_keys`.
3. השרת יוצר אתגר אקראי ומצפין אותו עם המפתח הציבורי שלך.
4. לקוח ה-SSH שלך מפענח את האתגר באמצעות המפתח הפרטי שלך.
5. הלקוח שולח הוכחה קריפטוגרפית (הנגזרת מהאתגר המפוענח ומזהה הסשן) בחזרה לשרת.
6. השרת מאמת את ההוכחה. רק מחזיק המפתח הפרטי יכול היה לייצר אותה.
7. האימות מצליח. שום סיסמה לא יוצאת אי פעם מהמכונה שלך.

זהו העיקרון המרכזי: **המפתח הפרטי שלך לעולם אינו עובר דרך הרשת**. השרת רואה רק את המפתח הציבורי שלך ואת ההוכחות הקריפטוגרפיות שאתה מייצר. גם אם מישהו יירט כל חבילה בסשן ה-SSH שלך, הוא לא מרוויח שום דבר שיעזור לו להתחזות אליך.

לרקע על פרוטוקול ה-SSH הרחב יותר, ראה את המדריך שלנו על <a href="/he/what-is-ssh/">מה זה SSH</a>.

## מפתח ציבורי מול מפתח פרטי: מה ההבדל?

הבנת ההבחנה בין שני המפתחות חיונית לשימוש נכון ב-SSH.

### המפתח הציבורי — המנעול שלך

**המפתח הציבורי** מיועד לשיתוף חופשי. חשוב עליו כמנעול: אתה מחלק עותקים לכולם במערכות שאתה רוצה לגשת אליהן. כשלשרת יש את המפתח הציבורי שלך בקובץ `authorized_keys` שלו, הוא יכול לאמת את זהותך מבלי לדעת את המפתח הפרטי שלך.

מפתחות ציבוריים:
- בטוחים לשיתוף — לא ניתן להשתמש בהם להתחזות אליך
- מוצבים בשרתים ב-`~/.ssh/authorized_keys`
- מועלים ל-GitHub, GitLab ושירותים אחרים
- בדרך כלל יש להם סיומת `.pub` (למשל, `id_ed25519.pub`)
- נראים כך: `ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAA... user@hostname`

### המפתח הפרטי — המפתח שלך

**המפתח הפרטי** הוא החצי הסודי של הזוג. הוא לעולם לא יכול לעזוב את המכונה שלך ואסור לשתפו עם אף אחד — לא עם מנהלים, לא ספקי ענן, לא עם צוות תמיכה. אם המפתח הפרטי שלך נפגע, כל מי שמשיג אותו יכול להתחזות אליך בכל שרת שבו מפתחך הציבורי מותקן.

מפתחות פרטיים:
- חייבים להישמר סודיים ומאובטחים
- מאוחסנים מקומית ב-`~/.ssh/` עם הרשאות מוגבלות (600)
- מוצפנים אופציונלית עם סיסמה
- ללא סיומת או עם סיומת ספציפית (למשל, `id_ed25519`)

### הקובץ `authorized_keys`

בצד השרת, המפתחות הציבוריים של כל משתמש מאוחסנים ב-`~/.ssh/authorized_keys`. זהו קובץ טקסט רגיל שבו כל שורה מכילה מפתח ציבורי אחד. כשלקוח SSH מנסה לאמת, השרת בודק אם הלקוח יכול להוכיח שהוא מחזיק במפתח הפרטי המתאים לאחד מהרשומות בקובץ זה.

```
# תבנית ~/.ssh/authorized_keys
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIGq... alice@laptop
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIHr... alice@workstation
ssh-rsa AAAAB3NzaC1yc2EAAAA... backup-deploy-key
```

ניתן לשים כמה מפתחות ציבוריים שצריך בקובץ זה, המאפשרים למספר מכשירים או מערכות אוטומציה לאמת לאותו שרת.

## אלגוריתמי מפתח SSH: Ed25519 מול RSA

כשאתה מייצר זוג מפתח SSH, אתה בוחר אלגוריתם. שתי האפשרויות הנפוצות ביותר כיום הן **Ed25519** ו-**RSA**. ישנם גם ECDSA ו-DSA, אך DSA הוצא משימוש ול-ECDSA יש מלכודות מימוש, אז נתמקד בשתי האפשרויות המומלצות.

### Ed25519 — הבחירה המודרנית

Ed25519 הוצג ל-OpenSSH בגרסה 6.5 (2014) והוא כיום ברירת המחדל המומלצת. הוא משתמש בקריפטוגרפיה של עקומה אליפטית על עקומת Edwards הפתולה (Curve25519), שתוכנן על ידי Daniel J. Bernstein.

**יתרונות Ed25519:**
- **קומפקטי** — מפתחות הם רק 68 תווים (מפתח ציבורי) והקבצים קטנים
- **מהיר** — יצירת חתימה ואימות מהירים ביותר
- **מאובטח בעיצוב** — אינו חשוף למתקפות ערוצי צד המשפיעות על חלק ממימושי RSA
- **גודל מפתח קבוע** — רמת אבטחה של 256 ביט, ללא צורך לבחור גודל מפתח
- **דטרמיניסטי** — החתימות דטרמיניסטיות, מה שמסיר סוג של כשלי מחולל מספרים אקראיים

```bash
# יצירת זוג מפתח Ed25519
ssh-keygen -t ed25519 -C "your@email.com"
```

### RSA — האפשרות האוניברסלית

RSA הוא האלגוריתם הישן יותר (הומצא ב-1977), ובעוד שהוא עדיין מאובטח ב-4096 ביט, הוא איטי יותר ומייצר מפתחות גדולים יותר מ-Ed25519. עם זאת, ל-RSA תאימות הרבה יותר רחבה — הוא עובד עם שרתי SSH ישנים מאוד, מערכות legacy וחלק ממפתחות אבטחת חומרה.

**מתי להשתמש ב-RSA:**
- אתה צריך להתחבר למערכות legacy עם גרסאות ישנות של OpenSSH (לפני 6.5)
- אתה משתמש בכרטיס חכם או HSM שאינו תומך ב-Ed25519
- נדרשת תאימות עם כלים ישנים

```bash
# יצירת זוג מפתח RSA בן 4096 ביט
ssh-keygen -t rsa -b 4096 -C "your@email.com"
```

**לעולם אל תשתמש ב-RSA עם פחות מ-4096 ביט.** RSA-1024 ו-RSA-2048 נחשבים יותר ויותר לבלתי מספיקים ככל שהכוח החישובי גדל. Ed25519 ב-256 ביט מספק אבטחה שקולה או טובה יותר עם חלק קטן מגודל המפתח.

### סיכום השוואה

| תכונה | Ed25519 | RSA 4096 |
|---|---|---|
| גודל מפתח (ציבורי) | ~68 תווים | ~725 תווים |
| רמת אבטחה | 128-ביט (מבנה עמיד לקוונטום) | ~140-ביט קלאסי |
| מהירות | מהיר מאוד | איטי יותר |
| תאימות | SSH מודרני (OpenSSH 6.5+) | אוניברסלית |
| מומלץ ל | כל המפתחות החדשים | מערכות legacy בלבד |

**שורה תחתונה: צור מפתחות Ed25519 לכל מקרה שימוש מודרני.** השתמש ב-RSA רק כשנכפה עליך בגלל דרישות תאימות.

## כיצד להוסיף מפתח SSH ל-GitHub

הוספת המפתח הציבורי שלך ל-GitHub מאפשרת לדחוף ולמשוך דרך SSH מבלי להזין שם משתמש וסיסמה בכל פעולה. הנה התהליך שלב-אחר-שלב.

### שלב 1: צור את זוג המפתחות

אם עדיין אין לך זוג מפתח SSH, צור אחד. ניתן להשתמש ב<a href="/">מחולל מפתחות SSH המקוון החינמי שלנו</a> או להריץ:

```bash
ssh-keygen -t ed25519 -C "your-github-email@example.com"
```

קבל את מיקום הקובץ הברירת מחדל (`~/.ssh/id_ed25519`) והגדר סיסמה אופציונלית.

### שלב 2: העתק את המפתח הציבורי שלך

הצג את המפתח הציבורי שלך והעתק אותו ללוח:

```bash
# Linux
cat ~/.ssh/id_ed25519.pub | xclip -selection clipboard

# macOS
cat ~/.ssh/id_ed25519.pub | pbcopy

# Windows (PowerShell)
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard
```

או פשוט הצג אותו והעתק ידנית:

```bash
cat ~/.ssh/id_ed25519.pub
```

### שלב 3: הוסף את המפתח ל-GitHub

1. התחבר ל-**GitHub.com**
2. לחץ על תמונת הפרופיל שלך ← **Settings**
3. בסרגל הצד השמאלי, לחץ **SSH and GPG keys**
4. לחץ **New SSH key**
5. תן לו כותרת תיאורית (למשל "MacBook Pro 2024" או "Work Laptop")
6. הדבק את המפתח הציבורי שלך בשדה **Key**
7. לחץ **Add SSH key**

### שלב 4: בדוק את החיבור

```bash
ssh -T git@github.com
```

אתה אמור לראות: `Hi username! You've successfully authenticated, but GitHub does not provide shell access.`

### מעבר מאגר קיים ל-SSH

אם שכפלת מאגר באמצעות HTTPS ורוצה לעבור ל-SSH:

```bash
# בדוק את כתובת ה-URL הנוכחית של ה-remote
git remote -v

# עדכן ל-SSH
git remote set-url origin git@github.com:username/repository.git
```

עבור GitLab, התהליך זהה — נווט אל **Preferences → SSH Keys** ועקוב אחרי אותם שלבים.

## איפה מפתחות SSH מאוחסנים?

מפתחות SSH מאוחסנים בתיקייה בשם `.ssh` בתיקיית הבית של המשתמש. המיקום משתנה לפי מערכת הפעלה.

### Linux ו-macOS

```
~/.ssh/
├── id_ed25519          # המפתח הפרטי שלך (הרשאות: 600)
├── id_ed25519.pub      # המפתח הציבורי שלך (הרשאות: 644)
├── id_rsa              # מפתח RSA פרטי (אם יש)
├── id_rsa.pub          # המפתח הציבורי של RSA
├── authorized_keys     # מפתחות ציבוריים המורשים להתחבר למכונה זו
├── known_hosts         # טביעות אצבע של שרתים שהתחברת אליהם
└── config              # קובץ תצורת לקוח
```

לתיקיית `~/.ssh/` עצמה חייבות להיות הרשאות **700** (קריאה/כתיבה/הרצה לבעלים בלבד). לקבצי מפתח פרטיים חייבות להיות הרשאות **600** (קריאה/כתיבה לבעלים בלבד). SSH יסרב להשתמש במפתחות עם הרשאות מאפשרות מדי.

```bash
# תיקון הרשאות במידת הצורך
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
chmod 644 ~/.ssh/id_ed25519.pub
```

### Windows

ב-Windows 10 ו-11 עם OpenSSH מותקן, מפתחות SSH מאוחסנים ב:

```
C:\Users\YourUsername\.ssh\
├── id_ed25519
├── id_ed25519.pub
├── authorized_keys
├── known_hosts
└── config
```

ב-Windows, ניתן לגשת לתיקיית `.ssh` מ-File Explorer (ייתכן שתצטרך להפעיל "הצג קבצים מוסתרים") או מ-PowerShell באמצעות `~\.ssh\`.

### קבצי מפתח מרובים

אם יש לך מפתחות למטרות שונות (GitHub אישי, GitLab לעבודה, שרתי ייצור), השתמש בקובץ `~/.ssh/config` כדי למפות כל מארח למפתחו:

```
Host github.com
    IdentityFile ~/.ssh/id_ed25519_github

Host gitlab.work.com
    IdentityFile ~/.ssh/id_ed25519_work

Host prod-server
    HostName 203.0.113.10
    User deploy
    IdentityFile ~/.ssh/id_ed25519_production
```

## סיסמאות למפתח SSH

כשאתה יוצר זוג מפתח SSH, `ssh-keygen` שואל אם ברצונך להגדיר **סיסמה**. זוהי סיסמה אופציונלית שמצפינה את קובץ המפתח הפרטי בדיסק.

### למה להשתמש בסיסמה?

אם קובץ המפתח הפרטי שלך מוגן בסיסמה חזקה, גניבת הקובץ אינה מספיקה להתחזות אליך — התוקף גם יצטרך לפצח את הסיסמה. זה מספק שכבת אבטחה שנייה קריטית, במיוחד על מחשבים ניידים שעלולים להיגנב פיזית.

ללא סיסמה, המפתח הפרטי שלך מאוחסן בטקסט רגיל בדיסק. כל מי שיכול לקרוא את הקובץ (דרך תוכנות זדוניות, גישה פיזית, או גיבוי שאינו מאובטח כראוי) יכול להשתמש בו מיד.

### כיצד סיסמאות מקיימות אינטראקציה עם ssh-agent

החיסרון של סיסמה הוא הקלדתה בכל פעם שאתה משתמש במפתח. **ssh-agent** פותר זאת על ידי החזקת מפתחות מפוענחים בזיכרון לאורך הסשן שלך.

```bash
# הפעל ssh-agent והוסף את המפתח שלך
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
# הזן סיסמה פעם אחת — מאוחסנת כעת לסשן
```

ב-macOS, ה-Keychain של המערכת מתחבר ל-ssh-agent. הוסף `-K` להמשכיות בין אתחולים מחדש:

```bash
ssh-add --apple-use-keychain ~/.ssh/id_ed25519
```

### שינוי סיסמה

ניתן לשנות את הסיסמה על מפתח קיים ללא יצירת זוג מפתחות מחדש:

```bash
ssh-keygen -p -f ~/.ssh/id_ed25519
```

זה שואל לסיסמה הישנה, ואז מאפשר לך להגדיר חדשה. המפתח הציבורי לא משתנה, כך שאינך צריך לעדכן אותו בשרתים כלשהם.

## טביעות אצבע של מפתחות SSH

לכל זוג מפתח SSH יש **טביעת אצבע** — גיבוב קצר של המפתח הציבורי שמזהה אותו באופן ייחודי. טביעות אצבע משמשות מספר מטרות:

- **אימות זהות מארח** — כשאתה מתחבר לראשונה לשרת, SSH מציג לך את טביעת האצבע של מפתח המארח שלו. אם אתה מאשר שהיא תואמת למה שציפית (למשל, שסופקה על ידי ספק האחסון שלך), אתה יודע שאתה מתחבר לשרת הנכון.
- **זיהוי מפתחות** — בניהול מפתחות רבים, טביעות אצבע עוזרות לזהות איזה מפתח פיזי תואם לאיזה רשומה ב-`authorized_keys` או בשירות כמו GitHub.
- **ביקורת וביטול** — יומני ביקורת לעתים קרובות מתעדים טביעות אצבע ולא מפתחות מלאים. ידיעת טביעת האצבע של מפתח עוזרת למצוא ולהסיר אותו אם צריך.

### צפייה בטביעת האצבע של מפתח

```bash
# צפייה בטביעת האצבע של המפתח הציבורי שלך
ssh-keygen -lf ~/.ssh/id_ed25519.pub
# פלט: 256 SHA256:abcdef1234567890... your@email.com (ED25519)

# צפייה בטביעת האצבע של מפתח המארח של שרת
ssh-keyscan server.example.com | ssh-keygen -lf -
```

GitHub מציג את טביעות האצבע של כל מפתחות ה-SSH שנוספו ב-**Settings → SSH and GPG keys**, המאפשר לאמת שהמפתח שאתה רואה שם תואם לזה שבמכונה שלך.

### הקובץ `known_hosts`

בפעם הראשונה שאתה מתחבר לשרת SSH חדש, אתה רואה הודעה כמו:

```
The authenticity of host 'example.com (203.0.113.10)' can't be established.
ED25519 key fingerprint is SHA256:abcdef1234567890...
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

הקלד `yes` וטביעת האצבע של השרת מתווספת ל-`~/.ssh/known_hosts`. בחיבורים עוקבים, SSH מאמת בשקט שמפתח השרת תואם לטביעת האצבע המאוחסנת. אם הם לא תואמים — מה שיכול להצביע על הגדרה שגויה או מתקפת איש-באמצע — SSH מזהיר אותך בקול ומסרב להתחבר.

## שאלות נפוצות

### למה משמש מפתח SSH?

מפתחות SSH מאמתים אותך לשרתים מרוחקים ללא סיסמה. השימושים הנפוצים ביותר הם: כניסה לשרתי Linux/Unix לניהול, אימות עם שירותי אירוח Git (GitHub, GitLab, Bitbucket) לגישת push/pull, סקריפטים לפריסה אוטומטית הדורשים גישה לשרת ללא סיסמה, וצינורות CI/CD הפורסים קוד לייצור. מפתחות SSH מועדפים על פני סיסמאות מכיוון שהם חסינים מפני כוח גסה, אינם ניתנים לשימוש חוזר בין פרצות, וניתן להשתמש בהם ללא אינטראקציה אנושית.

### כמה מפתחות SSH כדאי שיהיו לי?

אין תשובה אחת נכונה, אך שיטת העבודה המומלצת הנפוצה היא **מפתח אחד לכל מכשיר** (לא לכל שירות). למחשב הנייד שלך יש זוג מפתחות אחד; למחשב השולחני שלך יש אחר; לשרת CI שלך יש אחר. כך, אם מכשיר נגנב או נפגע, אתה מבטל רק את המפתח של אותו מכשיר. חלק מהאנשים מפצלים עוד יותר לפי הקשר (אישי מול עבודה), מה שמביא לשני מפתחות לכל מכשיר. הימנע משיתוף מפתח יחיד בין מספר מכונות — אם אותו מפתח נפגע, כל הגישה שלך בסיכון בו-זמנית.

### האם ניתן לשתף את המפתח הציבורי שלי ב-SSH?

כן. המפתח הציבורי מיועד לשיתוף. בטוח לשלוח אותו בדוא"ל, לפרסם אותו בתיעוד GitHub, או לכלול אותו בפנייה לגישה לשרת. המפתח הציבורי לבד לא ניתן להשתמש בו לאימות כאתה — זה דורש את המפתח הפרטי המתאים, שרק אתה מחזיק בו. זוהי התכונה הבסיסית של קריפטוגרפיה אסימטרית. לעולם אל תשתף את המפתח **הפרטי** שלך עם אף אחד.

### מה קורה אם אאבד את המפתח הפרטי שלי ב-SSH?

אם תאבד את המפתח הפרטי שלך ואין לך גיבוי, תאבד את היכולת לאמת איתו. המפתח הציבורי שלך, עדיין מותקן בשרתים ושירותים, הופך ליתום. תצטרך דרך חלופית לגשת לשרת (אימות סיסמה אם מופעל, גישת קונסול, מנגנון הגישה החירומי של ספק האחסון שלך) כדי להסיר את המפתח הציבורי הישן מ-`authorized_keys` ולהוסיף חדש. זו הסיבה שגיבוי המפתח הפרטי שלך (במנהל סיסמאות או גיבוי מוצפן) הוא חשוב. לא ניתן לשחזר את המפתח הפרטי מהמפתח הציבורי.

### האם Ed25519 או RSA עדיף?

**Ed25519 עדיף לכל יצירת מפתח חדשה.** הוא מהיר יותר, מייצר מפתחות קטנים יותר, אינו חשוף לחולשות מימוש מסוימות המשפיעות על RSA, ומספק אבטחה שקולה או עדיפה עם מפתח 256 ביט בהשוואה ל-RSA-4096. הסיבה היחידה להשתמש ב-RSA כיום היא תאימות עם תוכנת שרת SSH ישנה מאוד (OpenSSH לפני גרסה 6.5, שיצאה ב-2014) או מודולי אבטחת חומרה ישנים. אם אתה ניגש לכל שרת או שירות מודרני כמו GitHub, השתמש ב-Ed25519.

## צור את זוג מפתח ה-SSH שלך

מוכן ליצור את מפתחות ה-SSH שלך? המחולל המקוון החינמי שלנו יוצר זוגות מפתח Ed25519 או RSA ישירות בדפדפן שלך. המפתח הפרטי נוצר מקומית ולעולם אינו נשלח לשרתים שלנו.
