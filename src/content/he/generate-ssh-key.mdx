# יצירת מפתח SSH: כלי אונליין ופקודות ssh-keygen

יצירת מפתח SSH היא יצירת זוג של מפתח ציבורי ומפתח פרטי. למפתחות חדשים Ed25519 הוא בדרך כלל הבחירה הנוחה; RSA 4096 נדרש בעיקר לתאימות עם מערכות ישנות.

מחולל מפתחות SSH אונליין יוצר את זוג המפתחות בדפדפן. את המפתח הציבורי מוסיפים ל-GitHub, GitLab, Bitbucket או לשרת; המפתח הפרטי נשאר אצלכם במחשב.

## תשובה קצרה

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
```

<a href="/he/">מחולל מפתחות SSH: יצירת מפתחות אונליין</a> צרו מפתחות SSH מסוג Ed25519 או RSA בדפדפן עבור GitHub, GitLab, שרתים ו-ssh-keygen. המפתח הפרטי לא מועלה.

## יצירת מפתח SSH אונליין

1. פתחו את מחולל מפתחות ה-SSH.
2. בחרו Ed25519 למערכות חדשות או RSA 4096 לתאימות עם מערכות ישנות.
3. הוסיפו הערה ברורה, למשל אימייל או שם פרויקט.
4. למפתחות אישיים מומלץ להשתמש ב-passphrase.
5. צרו את זוג המפתחות ושמרו את שני הקבצים.
6. הוסיפו רק את המפתח הציבורי ל-GitHub, GitLab, Bitbucket או לשרת.

את המפתח הציבורי אפשר לשתף. המפתח הפרטי חייב להישאר סודי ולא להופיע בכרטיסים, צ׳אטים או תיעוד.

## יצירת מפתח SSH עם ssh-keygen

`ssh-keygen` / OpenSSH / Linux / macOS / Windows

```bash
ssh-keygen -t ed25519 -C "you@example.com"
ssh-keygen -t rsa -b 4096 -C "you@example.com"
ssh-keygen -t ed25519 -C "work@example.com" -f ~/.ssh/id_ed25519_work
```

## הוספת המפתח ל-GitHub, GitLab או שרת

הזרימה הרגילה: יוצרים מפתח, מעתיקים את המפתח הציבורי, מוסיפים אותו ל-GitHub/GitLab או לשרת, טוענים את הפרטי ל-ssh-agent ובודקים התחברות.

```bash
ssh -T git@github.com
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com
```

## בחירה בין Ed25519 ל-RSA

| שימוש | מפתח מומלץ |
|---|---|
| GitHub, GitLab, Bitbucket | Ed25519 |
| שרת Linux מודרני | Ed25519 |
| מפתח פריסה ל-CI/CD | Ed25519 |
| שרת SSH ישן | RSA 4096 |

למפתחות חדשים בחרו בדרך כלל Ed25519. השתמשו ב-RSA 4096 רק כששרת SSH ישן או כלי מסוים לא תומכים ב-Ed25519.

## Passphrase ו-ssh-agent

את המפתח הציבורי אפשר לשתף. המפתח הפרטי חייב להישאר סודי ולא להופיע בכרטיסים, צ׳אטים או תיעוד.

```bash
eval "$(ssh-agent -s)"
ssh-add ~/.ssh/id_ed25519
```

## פתרון בעיות נפוצות

ברוב החיבורים מספיק ssh user@host. השתמשו ב--i למפתח מסוים, ב--p לפורט אחר וב--J למעבר דרך bastion.

```bash
ssh -v user@server.example.com
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_ed25519
ssh -i ~/.ssh/id_ed25519_work user@server.example.com
```
