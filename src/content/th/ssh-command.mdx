# คำสั่ง SSH: การอ้างอิงที่สมบูรณ์พร้อมตัวอย่างสำหรับ Linux, Mac และ Windows

คำสั่ง `ssh` เป็นเครื่องมือหลักสำหรับการเริ่มต้นการเชื่อมต่อ SSH จากบรรทัดคำสั่ง ไม่ว่าคุณจะเข้าสู่ระบบเซิร์ฟเวอร์ Linux ระยะไกล ส่งต่อพอร์ตสำหรับการเข้าถึงฐานข้อมูล หรือกระโดดผ่านโฮสต์ป้อมปราการเพื่อเข้าถึงเครือข่ายภายใน คำสั่ง `ssh` คือจุดเริ่มต้นของคุณ คู่มือนี้ครอบคลุมทุกอย่างตั้งแต่ไวยากรณ์พื้นฐานไปจนถึงแฟล็กขั้นสูง การทันเนล และการกำหนดค่า ProxyJump — พร้อมตัวอย่างจริงสำหรับ Linux, macOS และ Windows

ก่อนที่จะรันคำสั่ง SSH คุณจะต้องมีคีย์ SSH ดูคำแนะนำของเราเกี่ยวกับ<a href="/th/what-is-an-ssh-key/">คีย์ SSH คืออะไร</a> หรือใช้<a href="/">โปรแกรมสร้างออนไลน์ฟรี</a>เพื่อสร้างคู่คีย์

## SSH พร้อมการรับรองความถูกต้องของคีย์: ธง -i

ตามค่าเริ่มต้น SSH จะค้นหาไฟล์คีย์ใน `~/.ssh/` ด้วยชื่อมาตรฐาน (`id_ed25519`, `id_rsa`, `id_ecdsa`, `id_dsa`) **`-i` flag** (ไฟล์ข้อมูลประจำตัว) ช่วยให้คุณสามารถระบุไฟล์คีย์อื่นได้อย่างชัดเจน

@@รหัส10@@

ธง `-i` มีความสำคัญในหลายสถานการณ์:
- คุณมีคู่คีย์หลายคู่สำหรับเซิร์ฟเวอร์หรือวัตถุประสงค์ที่แตกต่างกัน
- รหัสของคุณไม่อยู่ในไดเรกทอรีเริ่มต้น `~/.ssh/`
- คุณกำลังระบุคีย์ในสภาพแวดล้อม CI/CD ที่มีเส้นทางที่ชัดเจน
- คุณดาวน์โหลดคีย์ `.pem` จาก AWS หรือผู้ให้บริการคลาวด์รายอื่น

## ตัวเลือกคำสั่งและแฟล็กที่จำเป็นของ SSH

คำสั่ง `ssh` มีตัวเลือกมากมาย นี่คือสิ่งที่คุณจะใช้จริง

| ตั้งค่าสถานะ | แบบยาว / คำอธิบาย | ตัวอย่าง |
|---|---|---|
| `-p PORT` | เชื่อมต่อบนพอร์ตที่ไม่ได้มาตรฐาน | `ssh -p 2222 user@host` |
| `-i FILE` | ใช้ไฟล์ข้อมูลประจำตัว (คีย์) เฉพาะ | `ssh -i ~/.ssh/mykey user@host` |
| `-l USER` | ระบุชื่อผู้ใช้สำหรับเข้าสู่ระบบ (หรือแทน `user@`) | `ssh -l alice host` |
| `-L local:host:remote` | การส่งต่อพอร์ตท้องถิ่น | `ssh -L 8080:localhost:80 host` |
| `-R remote:host:local` | การส่งต่อพอร์ตระยะไกล | `ssh -R 9090:localhost:3000 host` |
| `-D port` | พร็อกซีไดนามิก (SOCKS) | `ssh -D 1080 host` |
| `-N` | อย่าดำเนินการคำสั่งระยะไกล (เฉพาะช่องทางทันเนลเท่านั้น) | `ssh -N -L 3307:db:3306 host` |
| `-f` | ไปที่พื้นหลังหลังจากการรับรองความถูกต้อง | `ssh -f -N -D 1080 host` |
| `-v` | เอาต์พุตแบบละเอียด (ดีบัก) | `ssh -v user@host` |
| `-vv` หรือ `-vvv` | เอาต์พุตแบบละเอียดเพิ่มเติม | `ssh -vvv user@host` |
| `-A` | เปิดใช้งานการส่งต่อตัวแทน SSH | `ssh -A user@jump-host` |
| `-X` | เปิดใช้งานการส่งต่อ X11 | `ssh -X user@host xclock` |
| `-C` | เปิดใช้งานการบีบอัด | `ssh -C user@host` |
| `-T` | ปิดใช้งานการจัดสรรเทอร์มินัลหลอก | `ssh -T git@github.com` |
| `-q` | โหมดเงียบ (ไม่แสดงคำเตือน) | `ssh -q user@host` |
| `-o OPTION=VALUE` | ตั้งค่าตัวเลือกการกำหนดค่าแบบอินไลน์ | `ssh -o StrictHostKeyChecking=no host` |
| `-J JUMP` | ProxyJump ผ่านโฮสต์ป้อมปราการ | `ssh -J jump.host user@internal` |
| `-W host:port` | ส่งต่อ stdio ไปยังโฮสต์: พอร์ต (สำหรับ ProxyJump) | ใช้ภายในโดย `-J` |

## SSH ProxyJump และโฮสต์ Bastion

ในหลายองค์กร เซิร์ฟเวอร์ที่ใช้งานจริงอยู่บนเครือข่ายส่วนตัวที่ไม่สามารถเข้าถึงได้โดยตรงจากอินเทอร์เน็ต **โฮสต์ป้อมปราการ** (หรือจัมเปอร์โฮสต์) คือเซิร์ฟเวอร์ที่เสริมความปลอดภัยบนขอบเขตเครือข่ายที่คุณ SSH เข้าก่อน จากนั้นจึงข้ามไปยังเซิร์ฟเวอร์ภายใน การตั้งค่าสถานะ `ssh -J` (ProxyJump ที่เพิ่มใน OpenSSH 7.3) ทำให้สิ่งนี้ราบรื่น

### ProxyJump เทียบกับการส่งต่อตัวแทน

แนะนำให้ใช้ ProxyJump มากกว่าการส่งต่อตัวแทน (`-A`) เนื่องจาก:
- รหัสส่วนตัวของคุณจะไม่ถูกส่งต่อไปยังโฮสต์ระดับกลาง
- โฮสต์ป้อมปราการไม่สามารถใช้ข้อมูลประจำตัวของคุณเพื่อตรวจสอบสิทธิ์ที่อื่นได้
- การเชื่อมต่อได้รับการรับรองความถูกต้องตั้งแต่ต้นทางถึงปลายทาง — เซิร์ฟเวอร์ปลายทางจะตรวจสอบคุณโดยตรง

## คำถามที่พบบ่อย

### ฉันจะรันคำสั่ง SSH โดยไม่ต้องใช้รหัสผ่านได้อย่างไร

ตั้งค่าการตรวจสอบสิทธิ์คีย์ SSH สร้างคู่คีย์ (`ssh-keygen -t ed25519`) คัดลอกคีย์สาธารณะไปยังไฟล์ `~/.ssh/authorized_keys` ของเซิร์ฟเวอร์โดยใช้ `ssh-copy-id user@host` และการเชื่อมต่อในภายหลังจะใช้คีย์ของคุณโดยไม่ต้องถามรหัสผ่าน หากคีย์ของคุณมีข้อความรหัสผ่าน ให้ใช้ `ssh-agent` และ `ssh-add` เพื่อแคชเพื่อให้คุณป้อนเพียงครั้งเดียวต่อเซสชัน

### ฉันจะออกจากเซสชัน SSH ได้อย่างไร

พิมพ์ `exit` หรือกด **Ctrl+D** เพื่อปิดเชลล์ระยะไกลและยกเลิกการเชื่อมต่อ หากเซสชันค้าง (เช่น เครือข่ายหลุด) ให้ใช้ลำดับการหลีกเลี่ยง SSH: กด **Enter** จากนั้นพิมพ์ `~.` (ตัวหนอนตามด้วยจุด) SSH ยุติการเชื่อมต่อทันที ลำดับการหลีกเลี่ยงอื่นๆ: `~^Z` ระงับ SSH ไว้ที่พื้นหลัง `~?` แสดงรายการลำดับการหลีกเลี่ยงทั้งหมด

สำหรับคำสั่งแบบโต้ตอบ (เช่น คำสั่งที่ต้องใช้เทอร์มินัล) ให้เพิ่มแฟล็ก `-t` เพื่อจัดสรรเทอร์มินัลเทียม: `ssh -t user@server sudo apt upgrade`

### ssh และ scp แตกต่างกันอย่างไร?

`ssh` เปิดเซสชันเทอร์มินัล (หรือรันคำสั่งเดียว) บนเครื่องระยะไกล `scp` (Secure Copy Protocol) คัดลอกไฟล์ระหว่างเครื่องของคุณและเครื่องระยะไกล หรือระหว่างเครื่องระยะไกลสองเครื่อง โดยใช้ SSH สำหรับการเข้ารหัสและการรับรองความถูกต้อง ทั้งสองใช้โปรโตคอล SSH และโครงสร้างพื้นฐานหลักเดียวกัน สำหรับการถ่ายโอนไฟล์ `scp` เป็นเครื่องมือเฉพาะ:

@@รหัส25@@

สำหรับการถ่ายโอนไฟล์ขั้นสูง (แบบเรียกซ้ำ ดำเนินการต่อ การเรียกดูไดเรกทอรี) ให้ใช้ `sftp` สำหรับการโอนเงินจำนวนมาก `rsync -avz -e ssh` จะมีประสิทธิภาพมากยิ่งขึ้น

### ฉันจะใช้ SSH กับไฟล์ปรับแต่งได้อย่างไร

สร้าง `~/.ssh/config` และเพิ่มรายการโฮสต์ แต่ละบล็อก `Host` กำหนดนามแฝงการเชื่อมต่อด้วยการตั้งค่าแบบเต็ม:

@@รหัส26@@

จากนั้น `ssh alias-name` จะใช้การตั้งค่าที่กำหนดไว้ทั้งหมดโดยอัตโนมัติ ไฟล์ปรับแต่งรองรับไวด์การ์ด: `Host *.example.com` ใช้การตั้งค่ากับโฮสต์ที่ตรงกันทั้งหมด `Host *` ในตอนท้ายจะใช้ค่าเริ่มต้นกับการเชื่อมต่อทั้งหมด ตัวเลือกในบล็อกเฉพาะ `Host` จะแทนที่ค่าเริ่มต้นส่วนกลาง

## ต้องการคีย์ SSH หรือไม่ สร้างได้ฟรี

หากต้องการใช้การตรวจสอบสิทธิ์คีย์ SSH ซึ่งไม่จำเป็นต้องใช้รหัสผ่าน คุณต้องมีคู่คีย์ สร้างคีย์ Ed25519 หรือ RSA ทันทีในเบราว์เซอร์ของคุณ โดยไม่ต้องส่งอะไรเลยไปยังเซิร์ฟเวอร์ของเรา
