# วิธีตั้งค่า SSH: คำแนะนำทีละขั้นตอนฉบับสมบูรณ์สำหรับ Linux, Mac และ Windows

การตั้งค่า SSH อย่างถูกต้องจะใช้เวลาประมาณสิบนาทีและจ่ายเงินปันผลเป็นเวลาหลายปี เมื่อกำหนดค่าแล้ว คุณสามารถเข้าสู่ระบบเซิร์ฟเวอร์โดยไม่ต้องพิมพ์รหัสผ่าน พุชโค้ดไปที่ GitHub โดยไม่ต้องป้อนข้อมูลประจำตัวในทุกการดำเนินการ และปรับใช้อัตโนมัติอย่างมั่นใจ คู่มือนี้จะอธิบายกระบวนการทั้งหมด ตั้งแต่การสร้างคู่คีย์ไปจนถึงการแก้ไขปัญหาข้อผิดพลาดทั่วไปสำหรับ Linux, macOS และ Windows

หากคุณต้องการทำความเข้าใจเทคโนโลยีที่สำคัญก่อน โปรดอ่านคำแนะนำของเราเกี่ยวกับ <a href="/th/what-is-ssh">SSH คืออะไร</a> และ <a href="/th/what-is-an-ssh-key">คีย์ SSH คืออะไร</a>

## ข้อกำหนดเบื้องต้น

ก่อนเริ่มต้น ตรวจสอบให้แน่ใจว่าคุณมี:

**สำหรับการเข้าถึงเซิร์ฟเวอร์:**
- แอปพลิเคชันเทอร์มินัล (เทอร์มินัลบน macOS/Linux, Windows Terminal หรือ PowerShell บน Windows)
- เซิร์ฟเวอร์ระยะไกลที่ใช้ Linux/Unix พร้อมติดตั้งซอฟต์แวร์เซิร์ฟเวอร์ SSH (sshd)
- บัญชีผู้ใช้บนเซิร์ฟเวอร์นั้น (หรือการเข้าถึงรูทเพื่อสร้างบัญชี)
- ที่อยู่ IP หรือชื่อโฮสต์ของเซิร์ฟเวอร์

**สำหรับการเข้าถึง GitHub/GitLab:**
- บัญชี GitHub หรือ GitLab
- มีเทอร์มินัลที่มี `ssh`
- ทางเลือก: ติดตั้ง git ในเครื่อง

**ความต้องการของระบบปฏิบัติการ:**
- **Linux**: ไคลเอนต์ SSH ได้รับการติดตั้งไว้ล่วงหน้าแล้วในแทบทุกการกระจาย
- **macOS**: ไคลเอ็นต์ SSH มีอยู่แล้วภายใน เข้าถึงได้จาก Terminal
- **Windows 10 เวอร์ชัน 1809+ / Windows 11**: ไคลเอนต์ OpenSSH มีให้เป็นคุณสมบัติเสริม

คุณไม่จำเป็นต้องติดตั้งอะไรบน Linux หรือ macOS บน Windows ให้ตรวจสอบว่าติดตั้ง OpenSSH แล้วโดยเปิด PowerShell และเรียกใช้ `ssh -V` หากคุณได้รับสตริงเวอร์ชัน แสดงว่าคุณพร้อมแล้ว ถ้าไม่ โปรดดูขั้นตอนที่ 4 สำหรับคำแนะนำในการติดตั้ง

## ขั้นตอนที่ 1: สร้างคู่คีย์ SSH ของคุณ

คุณต้องมีคู่คีย์ก่อนจึงจะสามารถดำเนินการอย่างอื่นได้ คู่คีย์ประกอบด้วย **คีย์ส่วนตัว** (อยู่บนเครื่องของคุณ) และ **คีย์สาธารณะ** (ใช้งานบนเซิร์ฟเวอร์หรือบริการ เช่น GitHub)

### ตัวเลือก A: ใช้ตัวสร้างออนไลน์

หากคุณต้องการอินเทอร์เฟซแบบกราฟิก <a href="/">ตัวสร้างคีย์ SSH ออนไลน์ฟรี</a>ของเราจะสร้างคู่คีย์ Ed25519 หรือ RSA ในเบราว์เซอร์ของคุณ รหัสส่วนตัวถูกสร้างขึ้นในเครื่องโดยใช้ Web Crypto API โดยไม่มีการส่งข้อมูลใดไปยังเซิร์ฟเวอร์ของเรา ดาวน์โหลดทั้งสองไฟล์และบันทึกไปที่ `~/.ssh/` (Linux/macOS) หรือ `C:\Users\YourName\.ssh\` (Windows)

### ตัวเลือก B: ใช้ ssh-keygen บน Command Line

คำสั่ง `ssh-keygen` พร้อมใช้งานบน Linux, macOS และ Windows ด้วย OpenSSH:

```bash
# สร้างคู่คีย์ Ed25519 (แนะนำ)
ssh-keygen -t ed25519 -C "your@email.com"
```

**ธงแต่ละอันหมายถึงอะไร:**
- `-t ed25519` — ประเภทคีย์ (Ed25519 ทันสมัย รวดเร็ว และปลอดภัย)
- `-C "comment"` — ป้ายกำกับที่ฝังอยู่ในกุญแจสาธารณะ ซึ่งโดยปกติจะเป็นที่อยู่อีเมลของคุณ

คุณจะได้รับแจ้งให้:

```
Generating public/private ed25519 key pair.
Enter file in which to save the key (/home/user/.ssh/id_ed25519):
```

กด **Enter** เพื่อยอมรับตำแหน่งเริ่มต้น หากคุณมีหลายคีย์ คุณสามารถตั้งชื่อคีย์เหล่านั้นโดยใช้คำอธิบายได้ (เช่น `/home/user/.ssh/id_ed25519_github`)

```
Enter passphrase (empty for no passphrase):
Enter same passphrase again:
```

ป้อนข้อความรหัสผ่านที่รัดกุม วิธีนี้จะเข้ารหัสไฟล์คีย์ส่วนตัวบนดิสก์ หากเครื่องของคุณถูกขโมย คุณจะไม่สามารถใช้คีย์ดังกล่าวได้หากไม่มีข้อความรหัสผ่าน คุณสามารถเว้นว่างไว้ได้สำหรับกรณีการใช้งานอัตโนมัติที่ไม่สามารถโต้ตอบกับมนุษย์ได้ แต่สำหรับคีย์ส่วนตัว ให้ใช้ข้อความรหัสผ่านเสมอ

**ผลลัพธ์:**

```
Your identification has been saved in /home/user/.ssh/id_ed25519
Your public key has been saved in /home/user/.ssh/id_ed25519.pub
The key fingerprint is:
SHA256:abc123defghijk... your@email.com
The key's randomart image is:
+--[ED25519 256]--+
...
```

มีการสร้างไฟล์สองไฟล์:
- `~/.ssh/id_ed25519` — **คีย์ส่วนตัว** ของคุณ (อย่าแชร์สิ่งนี้)
- `~/.ssh/id_ed25519.pub` — **กุญแจสาธารณะ** ของคุณ (ปลอดภัยสำหรับการแบ่งปัน)

### สำหรับ RSA (ความเข้ากันได้แบบเดิม)

หากคุณต้องการเชื่อมต่อกับระบบเก่ามากที่ไม่รองรับ Ed25519:

```bash
ssh-keygen -t rsa -b 4096 -C "your@email.com"
```

ใช้ขั้นต่ำ 4096 บิต คีย์ RSA ที่เล็กกว่า 4,096 บิตไม่ถือว่ามีความปลอดภัยเพียงพออีกต่อไป

## ขั้นตอนที่ 2: ตั้งค่า SSH บน Linux

ส่วนนี้ครอบคลุมถึงการคัดลอกคีย์สาธารณะของคุณไปยังเซิร์ฟเวอร์ Linux เพื่อให้คุณสามารถตรวจสอบได้โดยไม่ต้องใช้รหัสผ่าน

### วิธีที่ 1: ssh-copy-id (ง่ายที่สุด)

`ssh-copy-id` เป็นวิธีที่ง่ายที่สุดในการติดตั้งคีย์สาธารณะของคุณบนเซิร์ฟเวอร์ระยะไกล เชื่อมต่อโดยใช้การตรวจสอบรหัสผ่าน เพิ่มกุญแจสาธารณะของคุณไปที่ `~/.ssh/authorized_keys` ของเซิร์ฟเวอร์ และตั้งค่าการอนุญาตที่ถูกต้อง

```bash
ssh-copy-id user@server.example.com

# ระบุไฟล์คีย์อย่างชัดเจน
ssh-copy-id -i ~/.ssh/id_ed25519.pub user@server.example.com

# เชื่อมต่อบนพอร์ตที่ไม่ได้มาตรฐาน
ssh-copy-id -p 2222 user@server.example.com
```

คุณจะได้รับแจ้งให้ใส่รหัสผ่านเซิร์ฟเวอร์เป็นครั้งสุดท้าย หลังจากนี้ คุณสามารถเชื่อมต่อกับ `ssh user@server.example.com` โดยใช้รหัสของคุณ

### วิธีที่ 2: การคัดลอกด้วยตนเอง

หากไม่มี `ssh-copy-id` (พบได้ทั่วไปในระบบ BSD บางระบบหรือการติดตั้ง Linux ขั้นต่ำ):

```bash
# แสดงคีย์สาธารณะของคุณและคัดลอกเอาต์พุต
cat ~/.ssh/id_ed25519.pub

# จากนั้น SSH ไปยังเซิร์ฟเวอร์ (พร้อมรหัสผ่าน) และเรียกใช้:
mkdir -p ~/.ssh
chmod 700 ~/.ssh
echo "PASTE_YOUR_PUBLIC_KEY_HERE" >> ~/.ssh/authorized_keys
chmod 600 ~/.ssh/authorized_keys
```

หรือทำทุกอย่างในขั้นตอนเดียว:

```bash
cat ~/.ssh/id_ed25519.pub | ssh user@server.example.com \
  "mkdir -p ~/.ssh && chmod 700 ~/.ssh && \
   cat >> ~/.ssh/authorized_keys && chmod 600 ~/.ssh/authorized_keys"
```

### กำลังตรวจสอบการติดตั้งคีย์

```bash
ssh user@server.example.com
# ควรเชื่อมต่อโดยไม่ต้องถามรหัสผ่าน (เฉพาะข้อความรหัสผ่านหากคุณตั้งไว้)
```

### การกำหนดค่าเซิร์ฟเวอร์ SSH (sshd)

บนเซิร์ฟเวอร์ระยะไกล การกำหนดค่า SSH daemon อยู่ใน `/etc/ssh/sshd_config` เพื่อการตั้งค่าที่ปลอดภัย:

```bash
sudo nano /etc/ssh/sshd_config
```

การตั้งค่าที่แนะนำ:

```
# เปิดใช้งานการรับรองความถูกต้องของคีย์สาธารณะ
PubkeyAuthentication yes

# ปิดใช้งานการตรวจสอบรหัสผ่าน (หลังจากยืนยันการทำงานของคีย์แล้ว!)
PasswordAuthentication no

# ปิดใช้งานการเข้าสู่ระบบรูทผ่าน SSH
PermitRootLogin no

# อนุญาตเฉพาะผู้ใช้ที่ระบุเท่านั้น
AllowUsers alice bob deploy

# รักษาเซสชันให้คงอยู่
ClientAliveInterval 60
ClientAliveCountMax 3
```

หลังจากแก้ไขแล้ว ให้รีโหลด SSH daemon:

```bash
sudo systemctl reload sshd
# หรือบนระบบเก่า:
sudo service ssh reload
```

**สำคัญ**: ห้ามปิดการใช้งาน `PasswordAuthentication` จนกว่าคุณจะยืนยันว่าการตรวจสอบสิทธิ์แบบใช้คีย์ใช้งานได้ หากคุณล็อคตัวเองไว้ คุณจะต้องเข้าถึงคอนโซล (หรือการเข้าถึงฉุกเฉินของผู้ให้บริการโฮสต์ของคุณ) เพื่อกู้คืน

### การตั้งค่า ssh-agent บน Linux

หากคีย์ของคุณมีข้อความรหัสผ่าน ให้เพิ่มไปที่ `ssh-agent` เมื่อเริ่มต้นเซสชันของคุณเพื่อหลีกเลี่ยงการป้อนอีกครั้ง:

```bash
# เริ่มต้นเอเจนต์ (มักจะสตาร์ทอัตโนมัติโดยสภาพแวดล้อมเดสก์ท็อป)
eval "$(ssh-agent -s)"

# เพิ่มรหัสของคุณ (ป้อนรหัสผ่านหนึ่งครั้ง)
ssh-add ~/.ssh/id_ed25519

# ตรวจสอบคีย์ที่โหลดแล้ว
ssh-add -l
```

หากต้องการเริ่ม `ssh-agent` โดยอัตโนมัติและเพิ่มรหัสของคุณในการเข้าสู่ระบบ ให้เพิ่มไปที่ `~/.bashrc` หรือ `~/.zshrc`:

```bash
# เริ่มต้น ssh-agent อัตโนมัติ
if [ -z "$SSH_AUTH_SOCK" ]; then
  eval "$(ssh-agent -s)"
  ssh-add ~/.ssh/id_ed25519
fi
```

## ขั้นตอนที่ 3: ตั้งค่า SSH บน macOS

macOS มีไคลเอนต์ OpenSSH เต็มรูปแบบ กระบวนการในการตั้งค่าการตรวจสอบสิทธิ์แบบใช้คีย์ไปยังเซิร์ฟเวอร์ระยะไกลจะเหมือนกับ Linux (ดูขั้นตอนที่ 2 ด้านบน) การเพิ่มเฉพาะ macOS เกี่ยวข้องกับการรวมพวงกุญแจ

### บูรณาการพวงกุญแจ

macOS สามารถจัดเก็บข้อความรหัสผ่านของคุณในพวงกุญแจของระบบ ดังนั้นคุณไม่จำเป็นต้องป้อนรหัสผ่านดังกล่าวหลังจากครั้งแรก:

```bash
# เพิ่มคีย์ให้กับ ssh-agent ด้วยการรวมพวงกุญแจ
ssh-add --apple-use-keychain ~/.ssh/id_ed25519

# สำหรับ macOS เวอร์ชันเก่า (ก่อนมอนเทอเรย์)
ssh-add -K ~/.ssh/id_ed25519
```

หากต้องการให้ macOS โหลดคีย์ของคุณจากพวงกุญแจเสมอ ให้เพิ่มสิ่งนี้ไปที่ `~/.ssh/config`:

```
Host *
    AddKeysToAgent yes
    UseKeychain yes
    IdentityFile ~/.ssh/id_ed25519
```

หลังจากนี้ ข้อความรหัสผ่านจะถูกจัดเก็บไว้ในพวงกุญแจและโหลดโดยอัตโนมัติ คุณจะไม่ได้รับแจ้งให้ป้อนอีกครั้งในการบู๊ตครั้งต่อไป

### ตรวจสอบเวอร์ชันไคลเอ็นต์ SSH

```bash
ssh -V
# OpenSSH_9.x, LibreSSL x.x.x
```

macOS มาพร้อมกับ OpenSSH ที่ค่อนข้างใหม่ แต่ไม่ใช่รุ่นล่าสุดเสมอไป หากคุณต้องการคุณสมบัติใหม่ ให้ติดตั้งผ่าน Homebrew: `brew install openssh`

## ขั้นตอนที่ 4: ตั้งค่า SSH บน Windows

Windows 10 (1809+) และ Windows 11 มี OpenSSH นี่คือวิธีการทำให้มันทำงาน

### ติดตั้งไคลเอ็นต์ OpenSSH

**ตรวจสอบว่าติดตั้งแล้วหรือไม่:**

```powershell
ssh -V
```

**ติดตั้งผ่านการตั้งค่า:**
1. เปิด **การตั้งค่า → แอพ → คุณสมบัติเสริม**
2. ค้นหา **ไคลเอนต์ OpenSSH**
3. คลิก **ติดตั้ง**

**ติดตั้งผ่าน PowerShell (ผู้ดูแลระบบ):**

```powershell
Add-WindowsCapability -Online -Name OpenSSH.Client~~~~0.0.1.0
```

### สร้างคีย์บน Windows

เปิด PowerShell และเรียกใช้:

```powershell
ssh-keygen -t ed25519 -C "your@email.com"
```

กุญแจจะถูกบันทึกไปที่ `C:\Users\YourUsername\.ssh\`

### คัดลอกคีย์สาธารณะของคุณไปยังเซิร์ฟเวอร์ Linux

Windows ไม่มี `ssh-copy-id` แต่คุณสามารถทำได้ด้วยตนเองจาก PowerShell:

```powershell
# แสดงกุญแจสาธารณะของคุณ
Get-Content ~/.ssh/id_ed25519.pub

# คัดลอกไปยังเซิร์ฟเวอร์ระยะไกล (คุณจะได้รับแจ้งให้ใส่รหัสผ่านเซิร์ฟเวอร์)
$pubkey = Get-Content ~/.ssh/id_ed25519.pub -Raw
ssh user@server.example.com "mkdir -p ~/.ssh && echo '$pubkey' >> ~/.ssh/authorized_keys && chmod 700 ~/.ssh && chmod 600 ~/.ssh/authorized_keys"
```

หรือใช้ `Add-Content` เพื่อส่งผ่าน:

```powershell
type $env:USERPROFILE\.ssh\id_ed25519.pub | ssh user@server.example.com "cat >> ~/.ssh/authorized_keys"
```

### การตั้งค่า ssh-agent บน Windows

เพื่อหลีกเลี่ยงการป้อนข้อความรหัสผ่านซ้ำๆ ให้เริ่มบริการ OpenSSH Authentication Agent:

```powershell
# เริ่มบริการ
Start-Service ssh-agent

# ตั้งค่าให้เริ่มโดยอัตโนมัติ
Set-Service -Name ssh-agent -StartupType Automatic

# เพิ่มรหัสของคุณ
ssh-add ~/.ssh/id_ed25519
```

### PuTTY เป็นทางเลือก

หากคุณต้องการไคลเอนต์ SSH แบบกราฟิก PuTTY เป็นตัวเลือกฟรีสำหรับ Windows ใช้ **PuTTYgen** เพื่อสร้างคีย์ (บันทึกทั้งในรูปแบบ `.ppk` สำหรับ PuTTY และส่งออกในรูปแบบ OpenSSH) อย่างไรก็ตาม สำหรับการตั้งค่าใหม่ ไคลเอนต์ OpenSSH ในตัวจะง่ายกว่าและทำงานร่วมกับระบบนิเวศที่เหลือได้

## วิธีตั้งค่าคีย์ SSH สำหรับ GitHub และ GitLab

การตรวจสอบสิทธิ์คีย์ SSH ไปยัง GitHub และ GitLab ทำงานในลักษณะเดียวกัน — คุณอัปโหลดคีย์สาธารณะของคุณไปยังบริการ และใช้คีย์ส่วนตัวที่เกี่ยวข้องเมื่อดำเนินการ Git

### การตั้งค่า SSH สำหรับ GitHub

**1. คัดลอกกุญแจสาธารณะของคุณ:**

```bash
# ลินุกซ์
cat ~/.ssh/id_ed25519.pub | xclip -selection clipboard

# ระบบปฏิบัติการ macOS
cat ~/.ssh/id_ed25519.pub | pbcopy

# วินโดว์ PowerShell
Get-Content ~/.ssh/id_ed25519.pub | Set-Clipboard
```

หรือเพียงแค่แสดงและคัดลอกด้วยตนเอง:

```bash
cat ~/.ssh/id_ed25519.pub
```

**2. เพิ่มใน GitHub:**

1. ไปที่ **github.com** และเข้าสู่ระบบ
2. คลิกรูปประจำตัวของคุณ → **การตั้งค่า**
3. ในแถบด้านข้างซ้าย: **คีย์ SSH และ GPG**
4. คลิก **คีย์ SSH ใหม่**
5. ให้ **หัวข้อ** ที่สื่อความหมาย (เช่น "MacBook Pro" หรือ "Work PC")
6. ตั้งค่า **ประเภทคีย์** เป็น **คีย์การรับรองความถูกต้อง**
7. วางกุญแจสาธารณะของคุณ
8. คลิก **เพิ่มคีย์ SSH**

**3. ทดสอบการเชื่อมต่อ:**

```bash
ssh -T git@github.com
# สวัสดีชื่อผู้ใช้! คุณตรวจสอบสิทธิ์สำเร็จแล้ว แต่ GitHub ไม่ได้ให้การเข้าถึงเชลล์
```

**4. ใช้ SSH สำหรับการดำเนินการ Git:**

```bash
# โคลนโดยใช้ SSH แทน HTTPS
git clone git@github.com:username/repository.git

# หาก repo ที่มีอยู่ใช้ HTTPS ให้เปลี่ยนเป็น SSH
git remote set-url origin git@github.com:username/repository.git
```

### การตั้งค่า SSH สำหรับ GitLab

กระบวนการนี้เหมือนกัน:

1. ไปที่ **gitlab.com** → คลิกอวาตาร์ของคุณ → **การตั้งค่า**
2. ในแถบด้านข้างซ้าย: **คีย์ SSH**
3. วางคีย์สาธารณะของคุณ ตั้งชื่อ และอาจกำหนดวันหมดอายุก็ได้
4. คลิก **เพิ่มรหัส**

ทดสอบด้วย:

```bash
ssh -T git@gitlab.com
# ยินดีต้อนรับสู่ GitLab @ชื่อผู้ใช้!
```

สำหรับอินสแตนซ์ GitLab ที่โฮสต์เอง ให้แทนที่ `gitlab.com` ด้วยชื่อโฮสต์ของอินสแตนซ์ของคุณ

### การใช้คีย์แยกต่างหากสำหรับ GitHub และบริการอื่น ๆ

หากคุณต้องการคีย์เฉพาะสำหรับ GitHub (แนะนำเพื่อสุขอนามัยของคีย์) ให้สร้างคีย์ที่มีชื่อ:

```bash
ssh-keygen -t ed25519 -C "github" -f ~/.ssh/id_ed25519_github
```

จากนั้นเพิ่มบล็อก `Host` ไปที่ `~/.ssh/config`:

```
Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_ed25519_github
    AddKeysToAgent yes
```

ตอนนี้ `git clone git@github.com:...` จะใช้คีย์เฉพาะ GitHub โดยอัตโนมัติ

## การทดสอบและการแก้ไขปัญหา SSH

### การทดสอบการเชื่อมต่อขั้นพื้นฐาน

```bash
ssh -v user@server.example.com
```

ธง `-v` (verbose) แสดงให้เห็นอย่างชัดเจนถึงสิ่งที่เกิดขึ้นในแต่ละขั้นตอน มองหาบรรทัดเช่น:
- `Offering public key: ~/.ssh/id_ed25519` — SSH กำลังลองใช้คีย์ของคุณ
- `Server accepts key` — รหัสอยู่ใน `authorized_keys`
- `Authenticated to server.example.com` — สำเร็จ

### ข้อผิดพลาดและการแก้ไขทั่วไป

**"การอนุญาตถูกปฏิเสธ (คีย์สาธารณะ)"**

นี่เป็นข้อผิดพลาดที่พบบ่อยที่สุด สาเหตุและการแก้ไข:

```bash
# ตรวจสอบว่ารหัสของคุณโหลดอยู่ใน ssh-agent
ssh-add -l

# หากไม่มีคีย์อยู่ในรายการ ให้เพิ่มคีย์ของคุณ
ssh-add ~/.ssh/id_ed25519

# บนเซิร์ฟเวอร์ระยะไกล ให้ตรวจสอบauthorized_keys
cat ~/.ssh/authorized_keys
# ตรวจสอบให้แน่ใจว่ากุญแจสาธารณะของคุณอยู่ที่นั่น

# ตรวจสอบสิทธิ์บนเซิร์ฟเวอร์ระยะไกล
ls -la ~/.ssh/
# ~/.ssh ต้องเป็น 700
# Authorized_keys ต้องเป็น 600
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys

# ตรวจสอบว่าเซิร์ฟเวอร์อนุญาตการรับรองความถูกต้องของรหัสสาธารณะ
sudo grep PubkeyAuthentication /etc/ssh/sshd_config
# ต้องเป็น: PubkeyAuthentication ใช่
```

**"คำเตือน: การระบุโฮสต์ระยะไกลมีการเปลี่ยนแปลง"**

รหัสโฮสต์ของเซิร์ฟเวอร์มีการเปลี่ยนแปลง ซึ่ง SSH ถือเป็นปัญหาด้านความปลอดภัยที่อาจเกิดขึ้น หากคุณรู้ว่าสิ่งนี้ถูกต้อง (เซิร์ฟเวอร์ถูกสร้างขึ้นใหม่ มีการกำหนด IP ใหม่) ให้ลบรายการเก่าออก:

```bash
# ลบรายการโฮสต์เฉพาะ
ssh-keygen -R server.example.com

# หรือแก้ไขknown_hostsด้วยตนเองและลบบรรทัดที่ได้รับผลกระทบ
nano ~/.ssh/known_hosts
```

**"สิทธิ์ไม่ถูกต้อง" หรือ "สิทธิ์เปิดเกินไป"**

SSH เข้มงวดเกี่ยวกับการอนุญาตไฟล์คีย์:

```bash
# แก้ไขการอนุญาตคีย์ส่วนตัว
chmod 600 ~/.ssh/id_ed25519

# แก้ไขการอนุญาตคีย์สาธารณะ
chmod 644 ~/.ssh/id_ed25519.pub

# แก้ไขการอนุญาตไดเรกทอรี .ssh
chmod 700 ~/.ssh
```

บน Windows การอนุญาตคีย์ส่วนตัวจะได้รับการจัดการแตกต่างกัน คลิกขวาที่ไฟล์คีย์ → คุณสมบัติ → ความปลอดภัย → ขั้นสูง → ปิดการใช้งานการสืบทอด → ลบทั้งหมดยกเว้นบัญชีผู้ใช้ของคุณ โดยให้สิทธิ์การควบคุมทั้งหมดแก่ตัวคุณเองเท่านั้น

### การตั้งค่าและการใช้งาน ssh-agent

หากคุณได้รับแจ้งให้ป้อนข้อความรหัสผ่านซ้ำๆ `ssh-agent` ไม่ทำงานหรือไม่ได้เพิ่มรหัสของคุณ:

```bash
# เริ่มตัวแทน
eval "$(ssh-agent -s)"
# ตัวแทน p12345

# เพิ่มรหัสของคุณ
ssh-add ~/.ssh/id_ed25519
# ป้อนข้อความรหัสผ่านสำหรับ /home/user/.ssh/id_ed25519:
# เพิ่มข้อมูลประจำตัวแล้ว: /home/user/.ssh/id_ed25519

# ตรวจสอบ
ssh-add -l
# 256 SHA256:abc123... your@email.com (ED25519)
```

### การเติมข้อมูลknown_hostsล่วงหน้าในระบบอัตโนมัติ

ในไปป์ไลน์ CI/CD ให้ใช้ `ssh-keyscan` เพื่อเพิ่มลายนิ้วมือโฮสต์ระยะไกลก่อนที่จะเชื่อมต่อ:

```bash
ssh-keyscan -H server.example.com >> ~/.ssh/known_hosts
ssh-keyscan -H github.com >> ~/.ssh/known_hosts
```

วิธีนี้จะป้องกันการแจ้งลายนิ้วมือแบบโต้ตอบในสคริปต์อัตโนมัติ

## คำถามที่พบบ่อย

### ฉันจะแก้ไข "การอนุญาตถูกปฏิเสธ (คีย์สาธารณะ)" ได้อย่างไร

ดำเนินการตามรายการตรวจสอบนี้: (1) ตรวจสอบว่ารหัสของคุณอยู่ในตัวแทน SSH: `ssh-add -l` ถ้าไม่ ให้รัน `ssh-add ~/.ssh/id_ed25519` (2) ตรวจสอบว่ากุญแจสาธารณะของคุณอยู่ในไฟล์ `~/.ssh/authorized_keys` ของเซิร์ฟเวอร์ (3) ตรวจสอบการอนุญาต: `~/.ssh` ต้องเป็น `700`, `authorized_keys` ต้องเป็น `600` (4) ตรวจสอบว่าเซิร์ฟเวอร์มี `PubkeyAuthentication yes` ใน `/etc/ssh/sshd_config` (5) ใช้ `ssh -vvv user@server` เพื่อดูว่าการรับรองความถูกต้องล้มเหลวตรงไหน เอาต์พุตแบบละเอียดจะบอกคุณว่ากำลังลองใช้คีย์ใดและเซิร์ฟเวอร์ยอมรับคีย์เหล่านั้นหรือไม่

### ฉันจะเพิ่มคีย์ SSH ของฉันใน ssh-agent ได้อย่างไร

เรียกใช้ `ssh-add ~/.ssh/id_ed25519` (แทนที่ด้วยชื่อไฟล์คีย์ของคุณ) หากเอเจนต์ไม่ทำงาน ให้เริ่มต้นด้วย `eval "$(ssh-agent -s)"` ก่อน บน macOS ให้ใช้ `ssh-add --apple-use-keychain ~/.ssh/id_ed25519` เพื่อจัดเก็บข้อความรหัสผ่านในพวงกุญแจ บน Windows ให้เปิดใช้งานบริการ `ssh-agent` ด้วย `Start-Service ssh-agent` จากนั้นเรียกใช้ `ssh-add` คุณจะต้องเรียกใช้ `ssh-add` หนึ่งครั้งต่อเซสชัน (หรือหนึ่งครั้งด้วย Keychain บน macOS)

### ฉันสามารถมีคีย์ SSH หลายอันได้หรือไม่

ใช่แล้ว และถือเป็นแนวปฏิบัติที่ดี สร้างคีย์แยกกันเพื่อวัตถุประสงค์ที่แตกต่างกัน เช่น หนึ่งคีย์สำหรับ GitHub ส่วนตัว หนึ่งคีย์สำหรับ GitLab ที่ทำงาน หนึ่งคีย์สำหรับเซิร์ฟเวอร์ที่ใช้งานจริง ใช้ `~/.ssh/config` เพื่อแมปแต่ละโฮสต์กับคีย์เฉพาะด้วย `IdentityFile` การมีคีย์แยกกันหมายถึงการเพิกถอนคีย์หนึ่ง (โดยการลบออกจากบริการหรือเซิร์ฟเวอร์) จะไม่ส่งผลกระทบต่อคีย์อื่นๆ โดยจำกัดรัศมีการระเบิดของการประนีประนอมคีย์

### ฉันจะเปลี่ยนข้อความรหัสผ่านบนคีย์ SSH ได้อย่างไร

ใช้ `ssh-keygen -p`:

```bash
ssh-keygen -p -f ~/.ssh/id_ed25519
```

คุณจะได้รับแจ้งให้ป้อนรหัสผ่านปัจจุบัน จากนั้นให้ป้อนและยืนยันรหัสผ่านใหม่ ตัวกุญแจเอง (และกุญแจสาธารณะด้วย) จะไม่เปลี่ยนแปลง — คุณไม่จำเป็นต้องอัปเดตมันทุกที่ สิ่งนี้มีประโยชน์หากคุณสงสัยว่าข้อความรหัสผ่านของคุณถูกบุกรุกหรือต้องการหมุนเวียนเป็นแนวทางปฏิบัติด้านความปลอดภัยตามปกติ

### ฉันจะลบคีย์ SSH เก่าออกจากเซิร์ฟเวอร์ได้อย่างไร

เปิด `~/.ssh/authorized_keys` บนเซิร์ฟเวอร์และลบบรรทัดที่มีรหัสสาธารณะเก่า:

```bash
# แก้ไขauthorized_keysบนเซิร์ฟเวอร์ระยะไกล
ssh user@server.example.com
nano ~/.ssh/authorized_keys
# ลบบรรทัดสำหรับคีย์ที่คุณต้องการลบ
# บันทึกและออก
```

แต่ละบรรทัดใน `authorized_keys` คือคีย์สาธารณะหนึ่งคีย์ ความคิดเห็นของคีย์ (ฟิลด์สุดท้าย โดยปกติจะเป็น `user@hostname` หรืออีเมล) ช่วยระบุว่าคีย์ใดเป็นคีย์ใด หากไม่ทราบว่าควรลบบรรทัดไหน ให้เปรียบเทียบลายนิ้วมือ:

```bash
# บนเครื่องของคุณ ให้รับลายนิ้วมือ
ssh-keygen -lf ~/.ssh/id_ed25519.pub

# บนเซิร์ฟเวอร์ ให้ตรวจสอบลายนิ้วมือของคีย์ที่ได้รับอนุญาตทั้งหมด
ssh-keygen -lf ~/.ssh/authorized_keys
```

จับคู่ลายนิ้วมือเพื่อระบุและลบคีย์ที่ถูกต้อง

## สร้างคีย์ SSH ของคุณฟรี

หากคุณยังไม่ได้สร้างคู่คีย์ SSH โปรแกรมสร้างออนไลน์ฟรีของเราจะสร้างคีย์ Ed25519 หรือ RSA โดยตรงในเบราว์เซอร์ของคุณภายในไม่กี่วินาที ไม่จำเป็นต้องติดตั้ง ไม่มีการส่งข้อมูลใดๆ ไปยังเซิร์ฟเวอร์ของเรา — การสร้างเกิดขึ้นบนอุปกรณ์ของคุณทั้งหมด
